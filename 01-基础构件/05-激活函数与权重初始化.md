# 激活函数与权重初始化

## 历史背景：从阶跃函数到现代激活函数

### 1940-1960年代：阶跃函数的时代

1943年，McCulloch和Pitts提出的第一个神经元模型使用的是**阶跃函数**（Step Function）：

$$
f(x) = \begin{cases} 1 & x > 0 \\ 0 & x \leq 0 \end{cases}
$$

1957年，Rosenblatt的感知机也沿用了这个设计。阶跃函数简单直观——神经元要么"激活"（输出1），要么"沉默"（输出0），就像生物神经元的脉冲发放。

**但阶跃函数有一个致命问题：它不可微。**

在阶跃点，导数要么是0（平坦区域），要么是无穷大（跳跃点）。这意味着无法使用梯度下降来训练多层网络——你根本不知道该怎么调整权重。

### 1980年代：反向传播的复兴与Sigmoid的崛起

1986年，Rumelhart、Hinton和Williams发表了著名的论文，推广了**反向传播算法**（Backpropagation）。这篇论文证明了多层神经网络可以通过梯度下降来训练，重新点燃了神经网络研究的希望。

但反向传播需要一个**处处可微**的激活函数。研究者们找到了答案：**Sigmoid函数**。

$$
\sigma(x) = \frac{1}{1 + e^{-x}}
$$

Sigmoid的优点：

- 处处可微，导数简单：$\sigma'(x) = \sigma(x)(1-\sigma(x))$
- 输出范围(0,1)，可以解释为概率
- 平滑的S形曲线，符合直觉

**Sigmoid迅速成为1980-1990年代神经网络的标准选择。**

### 1990年代：梯度消失的困扰

然而，研究者们很快发现了Sigmoid的严重问题：**梯度消失**（Vanishing Gradient）。

考虑一个10层的网络，每层都使用Sigmoid。反向传播时，梯度要连续乘以每层的导数。而Sigmoid的导数最大值只有0.25：

$$
\sigma'(x) = \sigma(x)(1-\sigma(x)) \leq 0.25
$$

10层之后：$0.25^{10} \approx 0.00000095$

**梯度指数级衰减，底层的权重几乎收不到学习信号！**

这就是为什么1990年代的神经网络很难超过3-4层。这个问题直接导致了第二次AI寒冬——研究者们认为深度网络根本训练不了。

### 2000年代：Tanh的改进与ReLU的重新发现

**Tanh的改进**：研究者发现Tanh比Sigmoid好一些，因为它是零中心的（输出范围[-1,1]）。但Tanh仍然有梯度消失问题，只是程度稍轻。

**ReLU的重新发现**：2000年代初，一些研究者开始尝试一个极其简单的函数——**ReLU**（Rectified Linear Unit）：

$$
\text{ReLU}(x) = \max(0, x)
$$

ReLU其实早在1960年代就被提出过，但一直被忽视。直到2010年，Nair和Hinton、以及Glorot等人发表论文，证明ReLU在深层网络中效果惊人：

- 正区间梯度恒为1，彻底解决梯度消失
- 计算极其简单，只需判断正负
- 稀疏激活，一半神经元输出0

**ReLU成为深度学习复兴的关键之一。**

### 2010年代至今：激活函数的百花齐放

随着深度学习的爆发，研究者们不断改进ReLU：

- **Leaky ReLU**（2013）：解决"死亡ReLU"问题
- **GELU**（2016）：Transformer时代的宠儿
- **Swish**（2017）：Google用神经架构搜索发现的函数

## 数学符号说明

在深入学习激活函数之前，先了解一些常见的数学符号：

| 符号          | 名称                     | 含义                                                             |
| ------------- | ------------------------ | ---------------------------------------------------------------- |
| $e$         | 自然常数                 | 约等于 2.71828，自然对数的底数，指数函数的基础                   |
| $\sigma$    | Sigmoid 函数             | $\sigma(x) = \frac{1}{1+e^{-x}}$，输出范围(0,1)                |
| $\tanh$     | 双曲正切                 | $\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$，输出范围(-1,1) |
| $\Phi(x)$   | 标准正态分布累积分布函数 | 表示随机变量小于 x 的概率，用于 GELU                             |
| $n_{in}$    | 输入维度                 | 当前层神经元接收的输入数量                                       |
| $n_{out}$   | 输出维度                 | 当前层神经元输出的数量                                           |
| $\max(a,b)$ | 最大值函数               | 返回 a 和 b 中较大的那个值                                       |
| $\beta$     | 可学习参数               | 控制函数形状的参数，通常初始化为1                                |

**为什么用这些符号？**

- $e$：指数函数的基础，让 Sigmoid 和 Tanh 具有平滑的 S 形曲线
- $\sigma$：Sigmoid 的标准记号，形状像 S
- $\tanh$：双曲正切的缩写，比 Sigmoid 更对称
- $\Phi(x)$：统计学中的标准符号，表示"累积概率"
- $n_{in}$、$n_{out}$：描述网络层的维度，决定初始化的方差

## 1. 为什么需要激活函数

如果没有激活函数，无论多少层网络：

$$
y = W_3(W_2(W_1 x + b_1) + b_2) + b_3 = W_{combined} \cdot x + b_{combined}
$$

**结果仍然是线性的！** 本质上和单层网络一样。

激活函数引入**非线性**，让网络能够学习复杂模式。

## 2. 常见激活函数

### 2.1 Sigmoid

$$
\sigma(x) = \frac{1}{1 + e^{-x}}
$$

**特点**：

- 输出范围：(0, 1)
- 历史最悠久，早期神经网络的标准选择

**问题**：

- 梯度消失：$x$ 很大/很小时，梯度接近 0
- 输出非零中心：导致梯度更新效率低
- 计算慢：涉及指数运算

```
导数：σ'(x) = σ(x)(1 - σ(x))  最大值仅 0.25
```

### 2.2 Tanh

$$
\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}
$$

**特点**：

- 输出范围：(-1, 1)，零中心
- 比 Sigmoid 好，但仍有梯度消失问题

**关系**：$\tanh(x) = 2\sigma(2x) - 1$

**tanh(x) 的含义**：双曲正切函数，是 Sigmoid 的改进版本。

- 与 Sigmoid 的关系：$\tanh(x) = 2\sigma(2x) - 1$
- 输出范围：(-1, 1)，**零中心**（这是比 Sigmoid 好的地方）
- 形状也是 S 形，但关于原点对称
- 为什么零中心重要：Sigmoid 输出总是正数(0,1)，导致梯度更新效率低；Tanh 输出有正有负，梯度更新更高效

### 2.3 ReLU（最常用）

$$
\text{ReLU}(x) = \max(0, x)
$$

**特点**：

- 计算简单：只需判断正负
- 缓解梯度消失：正区间梯度恒为 1
- 稀疏激活：一半神经元输出 0

**问题**：

- Dead ReLU：如果神经元一直输出负值，梯度永远为 0，参数不再更新
- 输出非零中心

**max(0, x) 的含义**：取 0 和 x 中较大的那个值。

- 如果 x > 0：输出 x（保持原值）
- 如果 x ≤ 0：输出 0（截断为0）
- 直觉：ReLU就像一个"门"，只让正的信号通过，负的信号被完全阻断

### 2.4 Leaky ReLU

$$
\text{LeakyReLU}(x) = \begin{cases} x & x > 0 \\ \alpha x & x \leq 0 \end{cases}
$$

其中 $\alpha$ 通常取 0.01。

**优点**：负区间有小梯度，避免 Dead ReLU。

**α 参数的含义**：控制负区间的斜率，通常取 0.01。

- α = 0：就是标准 ReLU
- α > 0：负区间有小的正梯度，避免神经元"死亡"
- 直觉：给负信号留一条"活路"，不让梯度完全为0

### 2.5 GELU（Transformer 常用）

$$
\text{GELU}(x) = x \cdot \Phi(x)
$$

其中 $\Phi(x)$ 是标准正态分布的累积分布函数。

近似形式：

$$
\text{GELU}(x) \approx 0.5x \left(1 + \tanh\left[\sqrt{\frac{2}{\pi}}(x + 0.044715x^3)\right]\right)
$$

**特点**：

- BERT、GPT、ViT 等现代模型的默认选择
- 平滑的非线性，比 ReLU 更"柔和"

**Φ(x) 的含义**：标准正态分布的累积分布函数，表示"随机变量小于 x 的概率"。

- 当 x 很大时，Φ(x) ≈ 1，GELU(x) ≈ x（接近线性）
- 当 x 很小时，Φ(x) ≈ 0，GELU(x) ≈ 0（接近 ReLU）
- 当 x = 0 时，Φ(0) = 0.5，GELU(0) = 0（零中心）
- 直觉：GELU 根据输入的"大小"决定让多少信号通过，比 ReLU 更平滑

### 2.6 Swish / SiLU

$$
\text{Swish}(x) = x \cdot \sigma(\beta x)
$$

当 $\beta = 1$ 时，也称为 SiLU（Sigmoid Linear Unit）。

**特点**：

- Google 发现比 ReLU 在深层网络中表现更好
- EfficientNet 系列使用

**β 参数的含义**：可学习的参数，控制 Sigmoid 函数的"陡峭程度"。

- β 越大，σ(βx) 越接近阶跃函数，Swish 越接近 ReLU
- β 越小，σ(βx) 越平滑，Swish 越接近线性函数
- β = 1 时，就是 SiLU（Sigmoid Linear Unit）
- Google 的发现：让 β 作为可学习参数，网络可以自动找到最优的激活形状

## 4. 权重初始化的重要性

### 4.1 什么是权重初始化？

**权重初始化**（Weight Initialization）是指在训练开始前，为神经网络的权重参数赋予初始值的过程。

**为什么不能简单地设为0或相同的值？**

假设一个神经元有3个输入，权重都初始化为0.5：

- 输入：$x = [1, 2, 3]$
- 输出：$y = 0.5 \times 1 + 0.5 \times 2 + 0.5 \times 3 = 3$

**问题**：所有神经元输出相同，梯度也相同，学到的特征完全一样！这就是**对称性问题**（Symmetry Problem）。

**解决方案**：用**随机数**初始化，打破对称性，让每个神经元学到不同的特征。

**早期探索（1980-1990年代）**：研究者们用小随机数初始化（比如从均匀分布U(-0.1, 0.1)采样），或者用正态分布N(0, 0.01)。但问题在于：没有人知道"小"到底应该多小。初始化太小，信号逐层衰减，底层学不到东西；初始化太大，信号逐层放大，训练爆炸。研究者们只能凭经验调参，效果时好时坏。

### 4.2 为什么需要权重初始化？

**核心问题**：信号在深层网络中如何传播？

考虑一个5层网络，每层权重都初始化为0.1：

- 第1层输出：$y_1 = 0.1 \times x$
- 第2层输出：$y_2 = 0.1 \times y_1 = 0.01 \times x$
- 第5层输出：$y_5 = 0.1^5 \times x = 0.00001 \times x$

**结果**：信号衰减了100000倍！底层的权重几乎收不到学习信号。

反过来，如果权重都初始化为2：

- 第5层输出：$y_5 = 2^5 \times x = 32 \times x$

**结果**：信号放大了32倍，训练会爆炸（数值溢出）。

**这就是为什么需要精心设计初始化策略**：让信号既不衰减也不放大，稳定地在深层网络中传播。

### 4.3 目标：保持方差一致

2010年，Xavier Glorot和Yoshua Bengio发表了里程碑式的论文：

> *"Understanding the difficulty of training deep feedforward neural networks"*

这篇论文第一次从**方差分析**的角度系统研究了初始化问题。他们发现：

**核心洞察**：如果每一层的输出方差保持不变，信号就能稳定地在深层网络中传播。

这就是**Xavier初始化**的理论基础。他们还考虑了反向传播的梯度，最终给出的公式是：

$$
W \sim \mathcal{N}\left(0, \frac{2}{n_{in} + n_{out}}\right)
$$

**Xavier初始化让训练深层网络变得可行，是深度学习复兴的重要推动力之一。**


假设输入 $x$ 的方差是 $\text{Var}(x)$，我们希望每一层输出的方差保持不变。

对于线性层 $y = Wx$：

$$
\text{Var}(y) = n_{in} \cdot \text{Var}(W) \cdot \text{Var}(x)
$$

要让 $\text{Var}(y) = \text{Var}(x)$，需要：

$$
\text{Var}(W) = \frac{1}{n_{in}}
$$

### 4.4 He/Kaiming初始化适配ReLU（2015年）

2015年，何恺明（Kaiming He）等人发现Xavier初始化对ReLU不适用。

**原因**：ReLU把一半的输入变成0，相当于方差减半。如果还用Xavier的公式，信号会逐层衰减。

他们的解决方案很优雅：既然ReLU让方差减半，那就把初始化方差加倍：

$$
W \sim \mathcal{N}\left(0, \frac{2}{n_{in}}\right)
$$

这就是**He初始化**（也叫Kaiming初始化）。这篇论文还帮助他们赢得了ImageNet竞赛，证明了正确初始化的重要性。

### 4.5 现代：初始化已成为标准实践

今天，权重初始化已经不再是困扰研究者的问题。PyTorch、TensorFlow等框架都内置了这些初始化方法，研究者只需根据激活函数选择合适的初始化器即可。

## 5. 常见初始化方法

### 5.1 Xavier / Glorot 初始化

适用于 **Sigmoid / Tanh** 激活函数。

$$
W \sim \mathcal{N}\left(0, \frac{2}{n_{in} + n_{out}}\right)
$$

或均匀分布：

$$
W \sim \mathcal{U}\left(-\sqrt{\frac{6}{n_{in} + n_{out}}}, \sqrt{\frac{6}{n_{in} + n_{out}}}\right)
$$

**n_in 和 n_out 的含义**：

- n_in：输入维度，当前层神经元接收的输入数量
- n_out：输出维度，当前层神经元输出的数量
- 例如：nn.Linear(784, 256) 中，n_in = 784，n_out = 256
- Xavier 初始化同时考虑前向传播（n_in）和反向传播（n_out），取平均值

### 5.2 He / Kaiming 初始化

适用于 **ReLU** 激活函数。

$$
W \sim \mathcal{N}\left(0, \frac{2}{n_{in}}\right)
$$

**为什么是 2/n_in？** ReLU 把一半的值变成 0，方差减半，所以要乘以 2 补偿。

**n_in 的含义**：输入维度，即当前层神经元接收的输入数量。

- 例如：如果上一层有 784 个神经元，当前层有 256 个神经元，则 n_in = 784
- 初始化时需要根据 n_in 调整权重的方差，确保信号稳定传播
- n_in 越大，权重应该越小，避免信号放大

### 5.3 选择建议

| 激活函数          | 推荐初始化   |
| ----------------- | ------------ |
| Sigmoid / Tanh    | Xavier       |
| ReLU / Leaky ReLU | He (Kaiming) |
| GELU / Swish      | He (Kaiming) |

## 6. 代码示例

### 6.1 激活函数实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import matplotlib.pyplot as plt

# 绘制激活函数
x = torch.linspace(-5, 5, 100)

fig, axes = plt.subplots(2, 3, figsize=(12, 8))

activations = {
    'Sigmoid': torch.sigmoid(x),
    'Tanh': torch.tanh(x),
    'ReLU': F.relu(x),
    'LeakyReLU': F.leaky_relu(x, 0.1),
    'GELU': F.gelu(x),
    'SiLU/Swish': F.silu(x)
}

for ax, (name, y) in zip(axes.flat, activations.items()):
    ax.plot(x.numpy(), y.numpy())
    ax.set_title(name)
    ax.grid(True)
    ax.axhline(y=0, color='k', linewidth=0.5)
    ax.axvline(x=0, color='k', linewidth=0.5)

plt.tight_layout()
plt.savefig('activation_functions.png')
plt.show()
```

### 6.2 在模型中使用

```python
import torch.nn as nn

# 方式 1: 在 Sequential 中使用
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),           # 激活函数作为独立层
    nn.Linear(256, 10)
)

# 方式 2: 在 forward 中使用
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.fc2 = nn.Linear(256, 10)
  
    def forward(self, x):
        x = self.fc1(x)
        x = torch.relu(x)  # 函数式调用
        x = self.fc2(x)
        return x

# 方式 3: 使用 GELU（推荐用于 Transformer）
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.GELU(),
    nn.Linear(256, 10)
)
```

### 6.3 权重初始化

```python
import torch.nn as nn

def init_weights(module):
    if isinstance(module, nn.Linear):
        # He 初始化（适用于 ReLU）
        nn.init.kaiming_normal_(module.weight, mode='fan_in', nonlinearity='relu')
        if module.bias is not None:
            nn.init.zeros_(module.bias)
  
    elif isinstance(module, nn.Conv2d):
        nn.init.kaiming_normal_(module.weight, mode='fan_out', nonlinearity='relu')
        if module.bias is not None:
            nn.init.zeros_(module.bias)
  
    elif isinstance(module, nn.BatchNorm2d):
        nn.init.ones_(module.weight)
        nn.init.zeros_(module.bias)

# 应用初始化
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 128),
    nn.ReLU(),
    nn.Linear(128, 10)
)

model.apply(init_weights)

# 查看初始化后的权重统计
for name, param in model.named_parameters():
    if 'weight' in name:
        print(f"{name}: mean={param.mean():.4f}, std={param.std():.4f}")
```

### 6.4 PyTorch 默认初始化

PyTorch 的 `nn.Linear` 默认使用：

$$
W \sim \mathcal{U}\left(-\sqrt{\frac{1}{n_{in}}}, \sqrt{\frac{1}{n_{in}}}\right)
$$

这对于 ReLU 不是最优的，建议手动使用 He 初始化。

## 7. 常见问题

**Q: 为什么现代模型都用 GELU 而不是 ReLU？**

GELU 在零点附近更平滑，提供更好的梯度流动。实验表明在 Transformer 架构中 GELU 通常优于 ReLU。

**Q: 初始化时 bias 应该设成 0 吗？**

一般设为 0。但如果有 BatchNorm，最后一层的 bias 可以省略（被 BN 吸收）。

**Q: 预训练模型还需要关心初始化吗？**

从头训练需要。微调时一般保持预训练权重，只初始化新加的层。
