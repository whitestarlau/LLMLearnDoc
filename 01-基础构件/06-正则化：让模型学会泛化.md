# 正则化：让模型学会泛化

## 历史背景：为什么需要正则化？

### 1980-1990年代的困境

1986年，Rumelhart、Hinton和Williams发表了反向传播算法的著名论文，神经网络研究迎来了复兴。多层感知机（MLP）终于可以训练了！

但研究者们很快发现了新问题：

> **模型在训练集上表现完美，在新数据上却一塌糊涂——这就是过拟合。**

当时的计算机学者们面临一个核心困境：

- **小数据集**：标注数据昂贵，ImageNet（2009年）之前，大多数数据集只有几千张图片
- **大模型**：为了拟合复杂函数，网络越来越深，参数越来越多
- **过拟合**：模型把训练数据"背下来"了，而不是学到真正的规律

### 统计学的启示

正则化的思想其实源自更早的统计学。1970年，**Arthur Hoerl** 和 **Robert Kennard** 提出了**岭回归（Ridge Regression）**，通过在损失函数中加入权重的平方和来防止过拟合：

> *"Ridge regression: biased estimation for nonorthogonal problems"*

这就是今天L2正则化的前身。1996年，**Robert Tibshirani** 提出了**Lasso回归**，使用权重的绝对值之和，能够产生稀疏解——这就是L1正则化。

### 深度学习时代的创新

2012年，**Alex Krizhevsky**、**Ilya Sutskever** 和 **Geoffrey Hinton** 在AlexNet中首次使用了**Dropout**，将ImageNet分类错误率从26%降到15%。这个简单的技巧——训练时随机"关掉"一半神经元——震惊了整个领域。

2014年，**Nitish Srivastava** 等人发表了经典论文：

> *"Dropout: A Simple Way to Prevent Neural Networks from Overfitting"*

系统地证明了Dropout的有效性。

2015年，**Sergey Ioffe** 和 **Christian Szegedy** 提出了**Batch Normalization**，不仅加速了训练，还带来了意外的正则化效果。这篇论文成为深度学习历史上被引用最多的论文之一。

### 正则化的哲学

正则化的核心思想是**奥卡姆剃刀**：

> **在所有能解释数据的模型中，选择最简单的那个。**

因为简单的模型更可能学到真正的规律，而不是数据中的噪声。

## 1. 过拟合：模型的"死记硬背"

### 1.1 什么是过拟合

```
训练集表现：  ████████████████████ 99%
测试集表现：  ██████████░░░░░░░░░░ 70%
```

模型在训练数据上表现很好，但在新数据上表现很差——这就是**过拟合**。

### 1.2 直觉理解

想象一个学生：
- **欠拟合**：完全没学会，考试瞎猜
- **过拟合**：把练习题答案背下来了，但题目稍微变化就不会
- **泛化**：真正理解了知识点，任何变题都会做

正则化就是让模型"真正学会"而不是"死记硬背"。

## 2. 常见正则化方法

### 2.1 L2 正则化（权重衰减）

在损失函数中加入权重的 L2 范数：

$$
L_{total} = L_{data} + \frac{\lambda}{2} \sum_i w_i^2
$$

效果：让权重趋向于小的值，模型更"平滑"。

**直观理解**：小权重 = 简单模型 = 更好的泛化。

```python
# PyTorch 实现
optimizer = torch.optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-4)
```

### 2.2 L1 正则化

$$
L_{total} = L_{data} + \lambda \sum_i |w_i|
$$

效果：让部分权重变为 0，产生**稀疏模型**（自动特征选择）。

### 2.3 Dropout

训练时随机将一部分神经元的输出置为 0：

```
训练时：                    推理时：
[x] [x] [0] [x] [0]       [x] [x] [x] [x] [x]
 ↓   ↓   ↓   ↓   ↓         ↓   ↓   ↓   ↓   ↓
[h₁] [h₂] [0] [h₄] [0]   [h₁] [h₂] [h₃] [h₄] [h₅]
```

**为什么有效**：
- 防止神经元之间产生共依赖
- 相当于训练多个子网络的集成
- 迫使网络学习更鲁棒的特征

**注意**：推理时要乘以 $(1-p)$ 或使用 `model.eval()`。

```python
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.dropout = nn.Dropout(p=0.5)  # 50% 的概率丢弃
        self.fc2 = nn.Linear(256, 10)
    
    def forward(self, x):
        x = torch.relu(self.fc1(x))
        x = self.dropout(x)  # 训练时随机丢弃
        x = self.fc2(x)
        return x

# 使用
model.train()  # 启用 Dropout
output = model(data)

model.eval()   # 关闭 Dropout
with torch.no_grad():
    output = model(data)
```

### 2.4 Batch Normalization

对每个 mini-batch 进行归一化：

$$
\hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}
$$

$$
y = \gamma \hat{x} + \beta
$$

**效果**：
- 加速训练（允许更大学习率）
- 轻微正则化效果
- 减少对初始化的敏感度

```python
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.bn1 = nn.BatchNorm1d(256)
        self.fc2 = nn.Linear(256, 10)
    
    def forward(self, x):
        x = self.fc1(x)
        x = self.bn1(x)
        x = torch.relu(x)
        x = self.fc2(x)
        return x
```

### 2.6 Early Stopping

监控验证集损失，当不再下降时停止训练：

```
Epoch 1:  Train Loss 2.0, Val Loss 2.1
Epoch 10: Train Loss 0.5, Val Loss 0.6
Epoch 20: Train Loss 0.2, Val Loss 0.3  ← 继续下降
Epoch 30: Train Loss 0.1, Val Loss 0.29 ← 开始平稳
Epoch 40: Train Loss 0.05, Val Loss 0.31 ← 过拟合！停止
```

```python
best_val_loss = float('inf')
patience_counter = 0
patience = 10

for epoch in range(max_epochs):
    train_loss = train_epoch(model, train_loader)
    val_loss = validate(model, val_loader)
    
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        torch.save(model.state_dict(), 'best_model.pth')
        patience_counter = 0
    else:
        patience_counter += 1
    
    if patience_counter >= patience:
        print(f"Early stopping at epoch {epoch}")
        break

# 加载最佳模型
model.load_state_dict(torch.load('best_model.pth'))
```

### 2.8 Mixup

将两个样本混合：

```python
def mixup_data(x, y, alpha=0.2):
    lam = np.random.beta(alpha, alpha)
    batch_size = x.size(0)
    index = torch.randperm(batch_size)
    
    mixed_x = lam * x + (1 - lam) * x[index]
    y_a, y_b = y, y[index]
    return mixed_x, y_a, y_b, lam

# 训练时
mixed_x, y_a, y_b, lam = mixup_data(inputs, targets)
outputs = model(mixed_x)
loss = lam * criterion(outputs, y_a) + (1 - lam) * criterion(outputs, y_b)
```

## 3. 正则化方法选择指南

| 场景 | 推荐方法 |
|------|----------|
| 小数据集 | Dropout + 数据增强 + Early Stopping |
| CNN | BatchNorm + 数据增强 |
| Transformer | LayerNorm + Dropout |
| 非常深的网络 | BatchNorm + 权重衰减 |
| 过拟合严重 | 增大 Dropout 率 + 更强数据增强 |

## 4. 完整示例

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torchvision import datasets, transforms

class RegularizedMLP(nn.Module):
    def __init__(self, dropout_rate=0.3):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(784, 512),
            nn.BatchNorm1d(512),
            nn.ReLU(),
            nn.Dropout(dropout_rate),
            
            nn.Linear(512, 256),
            nn.BatchNorm1d(256),
            nn.ReLU(),
            nn.Dropout(dropout_rate),
            
            nn.Linear(256, 10)
        )
    
    def forward(self, x):
        return self.net(x.view(x.size(0), -1))

# 数据准备（带数据增强）
train_transform = transforms.Compose([
    transforms.RandomRotation(10),
    transforms.RandomAffine(degrees=0, translate=(0.1, 0.1)),
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,))
])

test_transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,))
])

train_dataset = datasets.MNIST('./data', train=True, download=True, transform=train_transform)
test_dataset = datasets.MNIST('./data', train=False, transform=test_transform)

train_loader = torch.utils.data.DataLoader(train_dataset, batch_size=64, shuffle=True)
test_loader = torch.utils.data.DataLoader(test_dataset, batch_size=1000)

# 模型与优化器（带 L2 正则化）
model = RegularizedMLP(dropout_rate=0.3)
optimizer = optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-4)
criterion = nn.CrossEntropyLoss()

# 训练与验证
for epoch in range(30):
    # 训练
    model.train()
    train_loss, correct, total = 0, 0, 0
    for data, target in train_loader:
        optimizer.zero_grad()
        output = model(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()
        
        train_loss += loss.item()
        pred = output.argmax(dim=1)
        correct += pred.eq(target).sum().item()
        total += target.size(0)
    
    # 验证
    model.eval()
    val_correct, val_total = 0, 0
    with torch.no_grad():
        for data, target in test_loader:
            output = model(data)
            pred = output.argmax(dim=1)
            val_correct += pred.eq(target).sum().item()
            val_total += target.size(0)
    
    print(f"Epoch {epoch+1}: Train Acc={correct/total:.4f}, Val Acc={val_correct/val_total:.4f}")
```

## 5. 常见问题

**Q: Dropout 率设多少合适？**

- 隐藏层：0.3 - 0.5
- 输入层：0.1 - 0.2
- 小数据集用大 Dropout，大数据集用小 Dropout

**Q: BatchNorm 和 Dropout 能一起用吗？**

可以，但要小心。BN 和 Dropout 的组合可能产生方差不匹配。实践建议：放在 BN 之后、ReLU 之前。

**Q: 大模型需要更强的正则化吗？**

大模型更容易过拟合，通常需要：
- 更大的数据集
- 更强的数据增强
- 更大的 Dropout 率
- Label Smoothing 等额外技巧

## 6. 小结

| 方法 | 原理 | 适用场景 |
|------|------|----------|
| L2 正则化 | 限制权重大小 | 通用 |
| Dropout | 随机失活神经元 | 全连接层 |
| BatchNorm | 归一化层输入 | CNN |
| LayerNorm | 归一化特征 | Transformer |
| Early Stopping | 监控验证集 | 通用 |
| 数据增强 | 增加数据多样性 | 图像、文本 |
