# 02-01 监督学习：有标签的学习

在前面的章节中，我们已经学会了如何构建神经网络、如何用反向传播训练它。但有一个关键问题我们一直没有深入讨论：**训练数据从哪里来？标签是什么？**

当你训练一个手写数字识别模型时，你告诉它"这张图是5"、"那张图是3"——这些"正确答案"就是标签。这种有标签的学习方式，就是**监督学习（Supervised Learning）**。

监督学习是深度学习中最常见、应用最广泛的范式。从图像分类到语音识别，从机器翻译到目标检测，绝大多数我们熟悉的AI应用都依赖监督学习。它的核心思想很简单：给模型看"问题和答案"，让它学会从输入预测输出。

但监督学习也有其局限性：标签从哪里来？标注成本有多高？当没有标签时怎么办？这些问题引出了后续的无监督学习和自监督学习。

本章将系统介绍监督学习的原理、常见任务类型和实践要点。

---

## 1. 监督学习的本质

### 1.1 什么是监督学习？

**监督学习 = 有标签的学习**

```
训练数据格式：
输入 (X)      标签 (Y)
─────────────────────────
图片1    →    "猫"
图片2    →    "狗"
图片3    →    "猫"
...

目标：学习一个函数 f: X → Y
     使得对新的输入 X_new，能预测正确的 Y_new
```

**"监督"的含义**：就像有老师监督学习一样，每个训练样本都有"正确答案"作为监督信号。模型预测错误时，标签会告诉它"你错了，正确答案是这个"。

### 1.2 与其他学习范式的对比

| 学习范式 | 数据特点 | 典型任务 | 例子 |
|---------|---------|---------|------|
| **监督学习** | 有标签 | 分类、回归 | 图像分类、房价预测 |
| **无监督学习** | 无标签 | 聚类、降维 | 客户分群、数据可视化 |
| **自监督学习** | 自动生成标签 | 预训练 | GPT预测下一个词 |
| **强化学习** | 奖励信号 | 决策优化 | 游戏AI、机器人控制 |

### 1.3 监督学习的两个主要任务

**分类任务（Classification）**：预测离散类别

```
例子：
- 图像分类：输入图片 → 输出类别（猫/狗/鸟）
- 垃圾邮件检测：输入邮件 → 输出（垃圾/正常）
- 情感分析：输入评论 → 输出（正面/负面）

损失函数：交叉熵损失
```

**回归任务（Regression）**：预测连续值

```
例子：
- 房价预测：输入房屋特征 → 输出价格
- 温度预测：输入历史数据 → 输出未来温度
- 股票预测：输入市场数据 → 输出股价

损失函数：均方误差（MSE）
```

## 2. 监督学习的完整流程

### 2.1 数据准备

```python
# 监督学习数据的典型结构
train_data = [
    (input_1, label_1),
    (input_2, label_2),
    (input_3, label_3),
    ...
]

# 例如：图像分类
# input: 图像的像素值 (224, 224, 3)
# label: 类别索引 (0-999)
```

**数据划分**：

```
完整数据集
    ├── 训练集 (70-80%) → 用于训练模型
    ├── 验证集 (10-15%) → 用于调参、选择模型
    └── 测试集 (10-15%) → 用于最终评估（只用一次）
```

### 2.2 模型训练

```python
import torch
import torch.nn as nn

# 1. 准备数据和标签
X = torch.randn(100, 10)  # 100个样本，每个10维
y = torch.randint(0, 2, (100,))  # 二分类标签

# 2. 定义模型
model = nn.Sequential(
    nn.Linear(10, 32),
    nn.ReLU(),
    nn.Linear(32, 2)
)

# 3. 定义损失函数和优化器
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

# 4. 训练循环
for epoch in range(100):
    # 前向传播
    outputs = model(X)
    loss = criterion(outputs, y)
    
    # 反向传播
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    if epoch % 20 == 0:
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")
```

### 2.3 评估指标

**分类任务指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| **准确率 (Accuracy)** | 正确预测数 / 总数 | 整体正确率 |
| **精确率 (Precision)** | TP / (TP + FP) | 预测为正的有多少是真的 |
| **召回率 (Recall)** | TP / (TP + FN) | 真正的有多少被找到 |
| **F1分数** | 2 × P × R / (P + R) | 精确率和召回率的调和平均 |

```python
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score

# 预测
y_pred = model(X).argmax(dim=1)

# 计算
accuracy = accuracy_score(y, y_pred)
precision = precision_score(y, y_pred)
recall = recall_score(y, y_pred)
f1 = f1_score(y, y_pred)
```

**回归任务指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| **MAE** | Σ\|y - ŷ\| / n | 平均绝对误差 |
| **MSE** | Σ(y - ŷ)² / n | 均方误差 |
| **RMSE** | √MSE | 均方根误差 |
| **R²** | 1 - SS_res/SS_tot | 决定系数 |

## 3. 监督学习的关键问题

### 3.1 过拟合与欠拟合

```
欠拟合 (Underfitting)：
- 模型太简单，学不到数据的规律
- 训练误差高，测试误差也高
- 解决：增加模型复杂度、增加特征

过拟合 (Overfitting)：
- 模型太复杂，记住了训练数据的噪声
- 训练误差低，测试误差高
- 解决：正则化、Dropout、增加数据、早停

理想状态：
- 训练误差和测试误差都低且接近
```

### 3.2 标签从哪里来？

**监督学习的关键瓶颈**：需要大量标注数据

```
标注来源：
1. 人工标注
   - 准确但昂贵
   - ImageNet：1400万张图片，人工标注3年
   
2. 众包标注
   - Amazon Mechanical Turk 等平台
   - 成本较低，质量控制是挑战
   
3. 自动标注
   - 利用现有工具或规则生成
   - 可能引入噪声

4. 半自动标注
   - 人工 + AI 辅助
   - 效率和质量的平衡
```

### 3.3 数据不平衡问题

```
问题：某些类别样本很少
例如：垃圾邮件检测（99%正常，1%垃圾）

解决方案：
1. 重采样
   - 过采样少数类（复制或生成）
   - 欠采样多数类（随机删除）

2. 类别权重
   loss = nn.CrossEntropyLoss(weight=torch.tensor([0.1, 0.9]))

3. 数据增强
   - 对少数类进行变换生成新样本

4. 换评估指标
   - 不用准确率，用F1分数或AUC
```

## 5. 监督学习的局限与展望

**优势**：
- 目标明确，训练高效
- 效果可预测，易于评估
- 应用场景广泛

**局限**：
- 需要大量标注数据（成本高）
- 只能学到"教给它的东西"
- 难以发现数据中的隐含模式
- 标签噪声会影响模型质量

**这引出了后续的学习范式**：
- **无监督学习**：没有标签时如何学习？
- **自监督学习**：如何自动生成监督信号？
- **强化学习**：如何从延迟的奖励中学习？
