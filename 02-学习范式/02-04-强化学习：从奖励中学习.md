# 02-04 强化学习：从奖励中学习

我们已经学习了三种学习范式：监督学习从标签中学习，无监督学习从数据结构中学习，自监督学习从自动生成的标签中学习。现在我们来学习第四种范式——**强化学习（Reinforcement Learning, RL）**。

强化学习与众不同：它从"奖励"中学习。就像训练一只小狗，你不会告诉它每个动作应该怎么走，而是在它做对了时给它零食，做错了时不给。小狗会逐渐学会哪些行为能获得奖励。

这种"试错学习"的方式让AI能够解决更复杂的问题：下围棋的AlphaGo、玩游戏的AI、控制机器人的策略，以及让GPT变成ChatGPT的RLHF——都依赖强化学习。

本章将介绍强化学习的核心概念、经典算法，以及它与深度学习的结合。

---

## 1. 强化学习的本质

### 1.1 什么是强化学习？

**强化学习 = 从奖励中学习最优行为策略**

```
监督学习：
 输入 → 模型 → 预测
             ↓
         与标签比较 → 计算损失 → 更新模型

强化学习：
 状态 → 智能体 → 动作 → 环境 → 新状态 + 奖励
    ↑___________________________________|
                 根据奖励调整策略
```

**核心概念**：

| 概念 | 含义 | 例子 |
|------|------|------|
| **智能体 (Agent)** | 做决策的实体 | 玩游戏的AI |
| **环境 (Environment)** | 智能体所处的世界 | 游戏本身 |
| **状态 (State)** | 当前情况描述 | 游戏画面 |
| **动作 (Action)** | 智能体的行为 | 按键：上/下/左/右 |
| **奖励 (Reward)** | 即时反馈信号 | 得分+10 |
| **策略 (Policy)** | 如何选择动作 | "看到敌人就射击" |

### 1.2 与其他学习范式的对比

```
监督学习：老师告诉你正确答案
  "这张图是猫"

无监督学习：自己找规律
  "这些数据可以分成3类"

自监督学习：自己出题自己答
  "下一个词是什么？答案是'很好'"

强化学习：尝试，看结果，调整
  动作A → 奖励+10 → "以后多做"
  动作B → 奖励-5  → "以后少做"
```

| 特性 | 监督学习 | 强化学习 |
|------|---------|---------|
| 反馈时机 | 立即 | 可能延迟 |
| 反馈类型 | 正确答案 | 奖励分数 |
| 数据来源 | 预先收集 | 交互产生 |
| 目标 | 预测准确 | 累积奖励最大化 |

### 1.3 强化学习的独特挑战

**延迟奖励（Credit Assignment Problem）**：

```
棋局：
第1步：下在左上角
第2步：下在右上角
...
第100步：赢了！

问题：是哪一步导致的胜利？
- 第1步可能很重要，但奖励在第100步才给
- 如何将最终奖励"分配"给之前每一步？
```

**探索 vs 利用（Exploration vs Exploitation）**：

```
探索：尝试新的动作，可能发现更好的策略
利用：使用已知最好的策略，保证稳定收益

例子：选餐厅
- 探索：去新餐厅，可能更好也可能更差
- 利用：去老餐厅，保证品质

好的策略需要在探索和利用之间平衡
```

## 2. 马尔可夫决策过程（MDP）

### 2.1 MDP 的数学描述

强化学习通常建模为马尔可夫决策过程：

```
MDP = (S, A, P, R, γ)

S: 状态空间（所有可能的状态）
A: 动作空间（所有可能的动作）
P: 状态转移概率 P(s'|s,a)
R: 奖励函数 R(s,a,s')
γ: 折扣因子（0 ≤ γ < 1）
```

**马尔可夫性**：下一状态只依赖当前状态和动作，与历史无关

```
P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, ...) 
    = P(s_{t+1} | s_t, a_t)
```

### 2.2 累积奖励与价值函数

**目标**：最大化累积奖励

$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + ... = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$

**为什么需要折扣因子 γ？**

```
γ = 0：只关心即时奖励（短视）
γ = 0.9：重视未来奖励（远见）
γ = 1：同等重视所有奖励（可能导致无限和）

实际中通常 γ = 0.99 或 0.999
```

**状态价值函数 V(s)**：在状态 s 下，期望获得的累积奖励

$$V(s) = \mathbb{E}[G_t | S_t = s]$$

**动作价值函数 Q(s,a)**：在状态 s 下采取动作 a，期望获得的累积奖励

$$Q(s,a) = \mathbb{E}[G_t | S_t = s, A_t = a]$$

## 3. 经典强化学习算法

### 3.1 Q-Learning：学习动作价值

**核心思想**：学习一个 Q 表，记录每个 (状态, 动作) 对的价值

```
Q 表：
         动作1   动作2   动作3   动作4
状态1     1.0     0.5     0.3     0.2
状态2     0.4     2.0     0.1     0.3
状态3     0.2     0.3     1.5     0.8

策略：在每个状态选择 Q 值最大的动作
```

**Q-Learning 更新公式**：

$$Q(s,a) \leftarrow Q(s,a) + \alpha [r + \gamma \max_{a'} Q(s',a') - Q(s,a)]$$

```python
import numpy as np

class QLearning:
    def __init__(self, n_states, n_actions, lr=0.1, gamma=0.99, epsilon=0.1):
        self.q_table = np.zeros((n_states, n_actions))
        self.lr = lr
        self.gamma = gamma
        self.epsilon = epsilon
    
    def get_action(self, state):
        """ε-贪婪策略：以 ε 概率探索，1-ε 概率利用"""
        if np.random.random() < self.epsilon:
            return np.random.randint(self.q_table.shape[1])
        return np.argmax(self.q_table[state])
    
    def update(self, state, action, reward, next_state):
        """Q-Learning 更新"""
        best_next = np.max(self.q_table[next_state])
        td_target = reward + self.gamma * best_next
        td_error = td_target - self.q_table[state, action]
        self.q_table[state, action] += self.lr * td_error

# 使用示例
agent = QLearning(n_states=10, n_actions=4)

for episode in range(1000):
    state = 0  # 初始状态
    
    while True:
        action = agent.get_action(state)
        # next_state, reward = env.step(action)  # 与环境交互
        next_state = (state + action) % 10
        reward = 1 if next_state == 5 else 0
        
        agent.update(state, action, reward, next_state)
        state = next_state
        
        if next_state == 5:  # 终止状态
            break
```

### 3.2 策略梯度：直接学习策略

**核心思想**：不学 Q 值，直接学习一个策略函数 π(a|s)

```
策略：状态 → 动作的概率分布

π(a|s) = P(选择动作 a | 当前状态 s)

例如：
状态 "看到敌人"
π(射击|看到敌人) = 0.8
π(躲避|看到敌人) = 0.1
π(逃跑|看到敌人) = 0.1
```

**策略梯度定理**：

$$\nabla J(\theta) = \mathbb{E}_{\pi} [\nabla \log \pi_\theta(a|s) \cdot Q(s,a)]$$

```python
import torch
import torch.nn as nn
import torch.optim as optim

class PolicyNetwork(nn.Module):
    def __init__(self, n_states, n_actions):
        super().__init__()
        self.fc1 = nn.Linear(n_states, 64)
        self.fc2 = nn.Linear(64, n_actions)
    
    def forward(self, x):
        x = torch.relu(self.fc1(x))
        return torch.softmax(self.fc2(x), dim=-1)

class Reinforce:
    """REINFORCE: 简单的策略梯度算法"""
    
    def __init__(self, n_states, n_actions, lr=0.01, gamma=0.99):
        self.policy = PolicyNetwork(n_states, n_actions)
        self.optimizer = optim.Adam(self.policy.parameters(), lr=lr)
        self.gamma = gamma
    
    def get_action(self, state):
        """根据策略采样动作"""
        state = torch.FloatTensor(state).unsqueeze(0)
        probs = self.policy(state)
        action = torch.multinomial(probs, 1).item()
        log_prob = torch.log(probs[0, action])
        return action, log_prob
    
    def update(self, trajectory):
        """
        trajectory: [(log_prob, reward), ...]
        """
        returns = []
        G = 0
        
        # 计算累积奖励
        for _, reward in reversed(trajectory):
            G = reward + self.gamma * G
            returns.insert(0, G)
        
        returns = torch.tensor(returns)
        returns = (returns - returns.mean()) / (returns.std() + 1e-8)  # 归一化
        
        # 计算损失
        loss = 0
        for (log_prob, _), G in zip(trajectory, returns):
            loss -= log_prob * G
        
        # 更新策略
        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()

# 使用示例
agent = Reinforce(n_states=4, n_actions=2)

for episode in range(1000):
    state = [0, 0, 0, 1]  # 初始状态
    trajectory = []
    
    for t in range(100):
        action, log_prob = agent.get_action(state)
        # next_state, reward = env.step(action)
        next_state = state  # 简化
        reward = 1 if action == 1 else 0
        
        trajectory.append((log_prob, reward))
        state = next_state
        
        if reward == 1:
            break
    
    agent.update(trajectory)
```

### 3.3 Actor-Critic：结合价值与策略

**核心思想**：同时学习策略（Actor）和价值函数（Critic）

```
Actor（策略网络）：决定采取什么动作
Critic（价值网络）：评估动作有多好

Critic 提供"基线"，降低方差，加速学习
```

```python
class ActorCritic(nn.Module):
    def __init__(self, n_states, n_actions):
        super().__init__()
        
        # 共享特征提取
        self.shared = nn.Sequential(
            nn.Linear(n_states, 64),
            nn.ReLU()
        )
        
        # Actor：输出动作概率
        self.actor = nn.Linear(64, n_actions)
        
        # Critic：输出状态价值
        self.critic = nn.Linear(64, 1)
    
    def forward(self, x):
        features = self.shared(x)
        action_probs = torch.softmax(self.actor(features), dim=-1)
        state_value = self.critic(features)
        return action_probs, state_value
```

## 4. 深度强化学习

### 4.1 DQN：深度Q网络

**问题**：Q-Learning 用表格存储 Q 值，状态空间大时不可行

**解决方案**：用神经网络近似 Q 函数

```python
import torch
import torch.nn as nn
import numpy as np
from collections import deque
import random

class DQN(nn.Module):
    def __init__(self, n_states, n_actions):
        super().__init__()
        self.fc1 = nn.Linear(n_states, 128)
        self.fc2 = nn.Linear(128, 128)
        self.fc3 = nn.Linear(128, n_actions)
    
    def forward(self, x):
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        return self.fc3(x)

class DQNAgent:
    def __init__(self, n_states, n_actions, gamma=0.99, lr=0.001, 
                 epsilon=1.0, epsilon_min=0.01, epsilon_decay=0.995):
        self.q_network = DQN(n_states, n_actions)
        self.target_network = DQN(n_states, n_actions)
        self.target_network.load_state_dict(self.q_network.state_dict())
        
        self.optimizer = torch.optim.Adam(self.q_network.parameters(), lr=lr)
        self.memory = deque(maxlen=10000)
        
        self.gamma = gamma
        self.epsilon = epsilon
        self.epsilon_min = epsilon_min
        self.epsilon_decay = epsilon_decay
        self.n_actions = n_actions
    
    def remember(self, state, action, reward, next_state, done):
        self.memory.append((state, action, reward, next_state, done))
    
    def get_action(self, state):
        if np.random.random() < self.epsilon:
            return np.random.randint(self.n_actions)
        
        state = torch.FloatTensor(state).unsqueeze(0)
        q_values = self.q_network(state)
        return q_values.argmax().item()
    
    def train(self, batch_size=32):
        if len(self.memory) < batch_size:
            return
        
        # 采样
        batch = random.sample(self.memory, batch_size)
        states, actions, rewards, next_states, dones = zip(*batch)
        
        states = torch.FloatTensor(states)
        actions = torch.LongTensor(actions)
        rewards = torch.FloatTensor(rewards)
        next_states = torch.FloatTensor(next_states)
        dones = torch.FloatTensor(dones)
        
        # 计算 Q 值
        current_q = self.q_network(states).gather(1, actions.unsqueeze(1))
        
        # 计算目标 Q 值（用目标网络）
        with torch.no_grad():
            next_q = self.target_network(next_states).max(1)[0]
            target_q = rewards + self.gamma * next_q * (1 - dones)
        
        # 损失
        loss = nn.MSELoss()(current_q.squeeze(), target_q)
        
        # 更新
        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()
        
        # 衰减探索率
        self.epsilon = max(self.epsilon_min, self.epsilon * self.epsilon_decay)
    
    def update_target(self):
        self.target_network.load_state_dict(self.q_network.state_dict())
```

**DQN 的关键技巧**：

```
1. 经验回放（Experience Replay）
   - 存储过去的经历
   - 随机采样训练
   - 打破数据相关性

2. 目标网络（Target Network）
   - 用一个单独的网络计算目标
   - 定期更新
   - 稳定训练

3. ε-贪婪探索
   - 开始时多探索
   - 逐渐增加利用
```

### 4.2 PPO：近端策略优化

PPO 是当前最流行的深度强化学习算法之一，也是 RLHF 的核心。

```python
class PPO:
    def __init__(self, policy, lr=3e-4, gamma=0.99, clip_epsilon=0.2):
        self.policy = policy
        self.optimizer = torch.optim.Adam(policy.parameters(), lr=lr)
        self.gamma = gamma
        self.clip_epsilon = clip_epsilon
    
    def compute_advantages(self, rewards, values, dones):
        """计算优势函数"""
        advantages = []
        G = 0
        
        for r, v, done in zip(reversed(rewards), reversed(values), reversed(dones)):
            if done:
                G = 0
            G = r + self.gamma * G
            advantages.insert(0, G - v)
        
        return advantages
    
    def update(self, states, actions, old_log_probs, advantages):
        for _ in range(10):  # 多次更新
            # 计算新的 log_prob
            action_probs, values = self.policy(states)
            dist = torch.distributions.Categorical(action_probs)
            new_log_probs = dist.log_prob(actions)
            
            # 重要性采样比率
            ratio = torch.exp(new_log_probs - old_log_probs)
            
            # PPO 裁剪目标
            surr1 = ratio * advantages
            surr2 = torch.clamp(ratio, 1 - self.clip_epsilon, 1 + self.clip_epsilon) * advantages
            loss = -torch.min(surr1, surr2).mean()
            
            # 更新
            self.optimizer.zero_grad()
            loss.backward()
            self.optimizer.step()
```

## 5. 强化学习与 RLHF

### 5.1 RLHF 中的强化学习

在 RLHF 中，我们用强化学习来优化语言模型：

```
RLHF 三阶段：
1. SFT（有监督微调）：让模型学会回答
2. RM（训练奖励模型）：学习人类偏好
3. PPO（强化学习）：用奖励模型优化策略

强化学习设定：
- 智能体：语言模型（策略）
- 环境：语言模型本身（生成文本）
- 动作：生成下一个 token
- 状态：当前的上下文
- 奖励：奖励模型的打分
```

### 5.2 为什么 RLHF 需要强化学习？

```
监督学习的局限：
- 只能学习"标准答案"
- 难以学习"什么是好的回答"

强化学习的优势：
- 可以优化复杂的目标（如"有帮助、无害、诚实"）
- 奖励模型提供"好"与"坏"的信号
- 策略梯度让模型朝着"更好"的方向优化
```
