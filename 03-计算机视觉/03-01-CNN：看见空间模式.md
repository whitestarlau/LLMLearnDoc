# CNN：看见空间模式

## 1. 为什么需要 CNN？

假设你有一张 28×28 的灰度图像，想用 MLP 处理它。

输入维度：28 × 28 = 784

如果第一层有 1000 个神经元，需要的参数量：

$$
784 \times 1000 = 784,000
$$

**问题**：
- 参数太多，容易过拟合
- 计算量大，训练慢
- **完全丢失了空间结构**——MLP 把图像展平成一维向量，像素之间的邻域关系消失了

我们需要一种能**直接处理二维结构**的网络。

---

## 2. 卷积：局部感受野

### 2.1 核心思想

人类看图像时，不是一次性看完整张图，而是**局部扫描**。

CNN 的核心思想：
- 每个神经元只看图像的一个小区域（**感受野**）
- 所有神经元**共享权重**（同一个卷积核在整张图上滑动）

### 2.2 数学定义

给定输入图像 $X$ 和卷积核 $K$，输出 $Y$：

$$
Y_{i,j} = \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} X_{i+m, j+n} \cdot K_{m,n} + b
$$

其中 $k$ 是卷积核大小（通常是 3×3 或 5×5）。

**直觉**：卷积核就像一个"滑动窗口"，在图像上滑动，每次计算一个小区域的加权和。

### 2.3 示例：边缘检测

一个 3×3 的边缘检测卷积核：

```
-1  -1  -1
-1   8  -1
-1  -1  -1
```

**效果**：
- 中心像素与周围像素差异大（边缘）→ 输出大的正值
- 中心像素与周围像素相似 → 输出接近 0

## 3. 卷积的三个关键参数

### 3.1 步长（Stride）

步长决定卷积核每次移动多少像素。

| 步长 | 效果 |
|------|------|
| 1 | 输出尺寸基本不变 |
| 2 | 输出尺寸减半 |

### 3.2 填充（Padding）

在输入图像周围补零，控制输出尺寸。

**Valid Padding**（无填充）：输出变小
**Same Padding**（补零）：输出尺寸 = 输入尺寸

输出尺寸公式：

$$
O = \frac{W - K + 2P}{S} + 1
$$

其中：
- $W$：输入尺寸
- $K$：卷积核大小
- $P$：填充大小
- $S$：步长

### 3.3 通道数（Channels）

- 输入：RGB 图像 → 3 通道
- 输出：可以有任意多个通道（每个通道一个卷积核）

**多通道卷积**：每个输出通道的卷积核也有多层（与输入通道数相同），然后求和。

## 4. 池化：下采样

### 4.1 为什么需要池化？

卷积提取特征后，特征图可能仍然很大。池化的作用：
- 减少计算量
- 增加感受野
- 提供一定的平移不变性

### 4.2 最大池化

取区域内的最大值：

```
输入 2×2 区域：
1  3
2  4

最大池化输出：4
```

### 4.3 平均池化

取区域内的平均值：

```
输入 2×2 区域：
1  3
2  4

平均池化输出：2.5
```

**实践**：最大池化更常用，因为它能保留最显著的特征。

## 5. 感受野：层层扩大

### 5.1 什么是感受野

**感受野**：输出特征图上的一个像素，能"看到"输入图像的多大区域。

### 5.2 计算感受野

两个 3×3 卷积层（步长=1）：
- 第一层感受野：3×3
- 第二层感受野：5×5（因为第一层的每个像素已经看到了 3×3 的区域）

**公式**（无池化）：

$$
RF_l = RF_{l-1} + (k_l - 1) \times \prod_{i=1}^{l-1} s_i
$$

**直觉**：堆叠多个小卷积核，等价于一个大卷积核，但参数更少。

| 方案 | 参数量 | 感受野 |
|------|--------|--------|
| 一个 5×5 卷积 | 25 | 5×5 |
| 两个 3×3 卷积 | 9+9=18 | 5×5 |

## 6. CNN 架构模式

### 6.1 经典模式

```
输入图像
    ↓
[卷积 → ReLU → 池化] × N
    ↓
展平（Flatten）
    ↓
全连接层 × M
    ↓
输出（分类/回归）
```

### 6.2 特征层级

| 层次 | 学到的特征 |
|------|-----------|
| 浅层 | 边缘、颜色、纹理 |
| 中层 | 形状、部件 |
| 深层 | 物体、语义 |

**直觉**：CNN 从低级到高级逐层抽象，就像人类视觉系统。

## 7. 代码示例

```python
import torch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        # 特征提取
        self.features = nn.Sequential(
            # 第一个卷积块：3通道 -> 32通道
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),  # 尺寸减半
            
            # 第二个卷积块：32通道 -> 64通道
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),  # 尺寸再减半
            
            # 第三个卷积块：64通道 -> 128通道
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),  # 尺寸再减半
        )
        
        # 分类器
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(128 * 4 * 4, 512),  # 假设输入是32×32
            nn.ReLU(),
            nn.Dropout(0.5),
            nn.Linear(512, num_classes)
        )
    
    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

# 使用示例
model = SimpleCNN(num_classes=10)
x = torch.randn(1, 3, 32, 32)  # 批次大小1，3通道，32×32图像
output = model(x)
print(f"输出形状: {output.shape}")  # torch.Size([1, 10])
```

## 8. 常见问题

**Q: 卷积和全连接有什么区别？**

| | 卷积层 | 全连接层 |
|--|--------|----------|
| 参数量 | 少（权重共享） | 多 |
| 空间结构 | 保留 | 丢失 |
| 适用场景 | 图像、网格数据 | 表格数据、特征向量 |

**Q: 为什么用 3×3 卷积而不是更大的？**

- 两个 3×3 = 一个 5×5 的感受野，但参数更少
- 更多非线性激活（每层一个 ReLU）
- VGGNet 验证了这个设计

**Q: 1×1 卷积有什么用？**

- 改变通道数（降维或升维）
- 跨通道的信息融合
- GoogLeNet/ResNet 中大量使用

## 9. 小结

| 概念 | 要点 |
|------|------|
| 卷积 | 局部连接 + 权重共享 |
| 池化 | 下采样，增加感受野 |
| 感受野 | 层层扩大，堆叠小卷积等价大卷积 |
| 架构模式 | 卷积块堆叠 → 全连接分类器 |
| 核心优势 | 参数少、保留空间结构、层级特征 |
