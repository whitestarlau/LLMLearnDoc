# 11-注意力机制：直接看全局

## 1. 为什么需要注意力？

### 1.1 RNN 的困境

上一章我们学习了 Seq2Seq：Encoder 将输入序列压缩成一个上下文向量，Decoder 从这个向量解码生成输出。

```
"我爱你" → [Encoder] → [0.2, 0.8, -0.3] → [Decoder] → "I love you"
```

这个固定长度的向量就像**信息瓶颈**：
- 句子短时，信息能够完整保留
- 句子一长，信息就会被"挤压"丢失
- 模型被迫把所有信息塞进一个向量里，长句翻译质量急剧下降

### 1.2 人类翻译的启示

想一想你自己是如何翻译的：

```
源句：The cat sat on the mat
翻译到 "mat" 时，你会回头看源句中的 "mat"
翻译到 "cat" 时，你会回头看源句中的 "cat"
```

**关键洞察**：每次生成一个词时，我们都会重新审视整个源句，决定当前最需要关注哪里。

这就是**注意力机制**的核心思想——不要试图用一个固定向量记住所有信息，而是让模型在每一步都能"看到"完整输入，自己决定该关注哪里。

## 2. Q / K / V：三个角色

注意力机制引入了三个核心概念，类比**图书馆查资料**：

| 概念 | 全称 | 类比 | 作用 |
|------|------|------|------|
| **Q** (Query) | 查询 | "我要找什么" | 当前位置的需求 |
| **K** (Key) | 键 | "我有什么标签" | 每个位置的标识 |
| **V** (Value) | 值 | "我的实际内容" | 每个位置的真正信息 |

**流程**：
```
1. 用 Q 去和所有 K 比较（计算相似度）
2. 得到一组权重（表示每个位置有多重要）
3. 用权重对 V 加权求和
```

**直觉**：
- Q 是"当前需要什么"
- K 是"我能提供什么标签"
- 匹配得越好（Q·K 越大），就越多地取那个位置的 V

## 3. 缩放点积注意力

### 3.1 点积：衡量相似度

两个向量的点积可以衡量相似度：
- 方向一致 → 点积大 → 相似
- 方向相反 → 点积小（负）→ 不相似

```
Score(Q, K) = Q · K = Σ(q_i * k_i)
```

### 3.2 缩放：为什么除以 √d_k

当向量维度 d_k 很大时，点积的值会很大，导致 Softmax 的输出接近 one-hot（梯度极小）。

**缩放因子**：`1 / √d_k`

```
Scaled Score = (Q · K) / √d_k
```

**为什么是 √d_k？**
- 假设 Q 和 K 的每个分量都服从 N(0,1)
- d_k 个分量相乘再求和，方差就是 d_k
- 除以 √d_k 后，方差回到 1

### 3.3 Softmax：转为概率分布

Softmax 把任意实数映射到 (0,1) 区间，且总和为 1：
```
softmax(x_i) = e^(x_i) / Σ(e^(x_j))
```

大的分数 → 接近 1（重点关注）
小的分数 → 接近 0（忽略）

### 3.4 完整公式

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

**步骤分解**：
```
1. QK^T        → 计算所有位置对的相似度矩阵 [seq_len × seq_len]
2. / √d_k      → 缩放，防止数值过大
3. softmax     → 转为概率分布（每行和为1）
4. × V         → 加权求和得到输出
```

## 4. 代码实现

### 4.1 基础版（直接实现）

```python
import torch
import torch.nn.functional as F

def attention(Q, K, V, mask=None):
    """
    Q: [batch, seq_len_q, d_k]
    K: [batch, seq_len_k, d_k]
    V: [batch, seq_len_k, d_v]
    mask: [batch, seq_len_q, seq_len_k] (可选)
    """
    d_k = Q.size(-1)
    
    # 1. 计算相似度
    scores = torch.matmul(Q, K.transpose(-2, -1))  # [batch, q, k]
    
    # 2. 缩放
    scores = scores / (d_k ** 0.5)
    
    # 3. 可选：mask（用于 Decoder 的因果注意力）
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    
    # 4. Softmax
    weights = F.softmax(scores, dim=-1)  # [batch, q, k]
    
    # 5. 加权求和
    output = torch.matmul(weights, V)  # [batch, q, d_v]
    
    return output, weights
```

### 4.2 PyTorch 内置版本

```python
import torch.nn as nn

# 使用 PyTorch 内置的注意力
multihead_attn = nn.MultiheadAttention(
    embed_dim=512,  # 嵌入维度
    num_heads=8,     # 头数（下一章讲）
    batch_first=True # batch 维度在前
)

# 输入：Q, K, V 都是 [batch, seq_len, embed_dim]
output, attn_weights = multihead_attn(Q, K, V)
```

## 5. 自注意力 vs 交叉注意力

### 自注意力（Self-Attention）

Q、K、V 来自**同一个序列**：
```python
Q = K = V = input_sequence  # 都是同一个
output, weights = attention(Q, K, V)
```

**用途**：让序列中的每个位置都能看到其他所有位置
- BERT 的每一层
- GPT 的每一层

### 交叉注意力（Cross-Attention）

Q 来自一个序列，K、V 来自另一个序列：
```python
Q = decoder_output    # 解码器的输出
K = V = encoder_output  # 编码器的输出
output, weights = attention(Q, K, V)
```

**用途**：让一个序列"查询"另一个序列的信息
- 机器翻译：解码器查编码器
- 图像描述：文本查图像特征

## 6. 可视化理解

### 注意力权重矩阵

输入序列：`"The cat sat on the mat"`

注意力权重（一个位置对其他位置的关注程度）：
```
        The  cat  sat  on  the  mat
  The  [0.1  0.3  0.1 0.1  0.2  0.2]  ← "The" 主要看 cat, the, mat
  cat  [0.1  0.6  0.1 0.05 0.05 0.1]  ← "cat" 主要看自己
  sat  [0.1  0.4  0.3 0.1  0.05 0.05] ← "sat" 主要看 cat 和自己
  ...
```

每行之和 = 1（概率分布）

## 7. 为什么注意力有效？

### 7.1 全局视野

每个位置都能直接看到序列中的任意位置，不受距离限制。

### 7.2 动态权重

权重是根据输入动态计算的，不是固定的。
- 同一个词在不同语境中会关注不同的上下文

### 7.3 可并行

注意力计算不依赖前一步的结果，可以**完全并行**（对比 RNN 必须串行）。

### 7.4 梯度路径短

信息传递路径只有一步，梯度更容易流动。

## 8. 常见问题

### Q1: 为什么需要除以 √d_k 而不是 d_k？

除以 d_k 会让数值过小，softmax 输出接近均匀分布（失去区分度）。

### Q2: 注意力的时间复杂度是多少？

O(n² × d)，其中 n 是序列长度，d 是维度。这就是为什么长序列处理是挑战。

### Q3: Q、K、V 是怎么来的？

从输入向量通过不同的线性变换得到：
```python
Q = input @ W_Q  # [seq, d] @ [d, d_k] = [seq, d_k]
K = input @ W_K
V = input @ W_V
```

W_Q、W_K、W_V 是可学习的参数矩阵。

### Q4: Attention 和 Softmax 是一回事吗？

不是。Attention 是整体机制，Softmax 只是其中一步（将分数转为概率）。

## 9. 核心代码总结

```python
# 注意力的本质：加权求和
def simple_attention(query, keys, values):
    # 1. 计算相似度
    scores = query @ keys.T  # 点积
    # 2. 缩放
    scores = scores / sqrt(d_k)
    # 3. 转为概率
    weights = softmax(scores)
    # 4. 加权求和
    return weights @ values

# 一句话：用 query 找最相关的 key，然后取对应的 value
```
