# 12-多头注意力：并行的视角

## 1. 为什么需要多头？

### 1.1 单头注意力的局限

上一章我们学习了注意力机制：每个位置通过一组 Q/K/V 计算与其他位置的关系。

但当模型处理一个词时，它需要关注的信息往往是**多维度的**。例如：

```
"The cat sat on the mat"

当处理 "mat" 时，它应该关注什么？
- 句法角度：关注 "on"（介词搭配）
- 语义角度：关注 "cat"（谁坐在上面）
- 指代角度：关注 "the"（特指哪个mat）
```

**单个注意力头只能学到一种关系模式**，被迫在所有可能的关联中做取舍。

### 1.2 人类阅读的启示

想象多位专家同时阅读同一段话：
- 语法专家：关注主谓宾结构
- 语义专家：关注词义关联
- 位置专家：关注相邻关系

每位专家从自己的角度理解文本，最后综合所有视角，才能得到完整的理解。

**多头注意力的核心思想**：与其让一个头被迫做取舍，不如用多个头分别关注不同方面，再把结果整合。

---

## 2. 拆分：把一个大注意力拆成多个小注意力

### 2.1 核心思想

不是计算一个 512 维的注意力，而是计算 8 个 64 维的注意力：

```
原来的：1 个头，d_model = 512
拆分后：8 个头，每个头 d_k = 512 / 8 = 64
```

### 2.2 拆分方式

```python
# 原始 Q: [batch, seq_len, 512]
# 拆成 8 个头：每个头是 [batch, seq_len, 64]

# 方法1：直接 reshape
Q = Q.reshape(batch, seq_len, num_heads, d_k)  # [b, s, 8, 64]
Q = Q.transpose(1, 2)  # [b, 8, s, 64] 便于并行计算

# 方法2：先切分再 reshape
Q_chunks = Q.chunk(num_heads, dim=-1)  # 8个 [b, s, 64]
```

### 2.3 线性变换的实现

实际上，我们用**一个大矩阵**同时计算所有头：

```python
# 不是 8 个独立的线性层
# 而是一个大的线性层，输出后 reshape
W_Q = nn.Linear(d_model, d_model)  # 512 → 512
Q = W_Q(input)  # [batch, seq_len, 512]
Q = Q.view(batch, seq_len, num_heads, d_k)  # [b, s, 8, 64]
```

**为什么这样做？** 计算效率更高，GPU 更擅长大矩阵运算。

---

## 3. 拼接：把多个头的结果合起来

### 3.1 独立计算每个头

每个头独立做注意力计算：
```python
for i in range(num_heads):
    head_i = attention(Q_heads[i], K_heads[i], V_heads[i])
    # 每个 head_i: [batch, seq_len, 64]
```

### 3.2 拼接回原始维度

```python
# 拼接 8 个头
output = concat(head_1, head_2, ..., head_8)  # [batch, seq_len, 512]
```

### 3.3 线性投影

最后一层线性变换，混合不同头的信息：
```python
W_O = nn.Linear(d_model, d_model)
final_output = W_O(output)
```

---

## 4. 完整公式

```
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) × W_O

其中 head_i = Attention(Q × W_Qi, K × W_Ki, V × W_Vi)
```

**参数矩阵**：
- W_Q: [d_model, d_model] → 拆成 h 个 [d_model, d_k]
- W_K: [d_model, d_model] → 拆成 h 个 [d_model, d_k]
- W_V: [d_model, d_model] → 拆成 h 个 [d_model, d_k]
- W_O: [d_model, d_model] → 混合不同头的输出

---

## 5. 代码实现

### 5.1 完整实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0, "d_model 必须能被 num_heads 整除"
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # 每个头的维度
        
        # 线性投影层
        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        self.W_O = nn.Linear(d_model, d_model)
    
    def forward(self, Q, K, V, mask=None):
        batch_size = Q.size(0)
        
        # 1. 线性投影
        Q = self.W_Q(Q)  # [b, seq_q, d_model]
        K = self.W_K(K)  # [b, seq_k, d_model]
        V = self.W_V(V)  # [b, seq_k, d_model]
        
        # 2. 拆分成多个头
        Q = Q.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        # 现在是 [b, num_heads, seq_len, d_k]
        
        # 3. 计算注意力
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        weights = F.softmax(scores, dim=-1)
        attn_output = torch.matmul(weights, V)  # [b, num_heads, seq_q, d_k]
        
        # 4. 拼接所有头
        attn_output = attn_output.transpose(1, 2).contiguous()
        attn_output = attn_output.view(batch_size, -1, self.d_model)
        
        # 5. 最终线性投影
        output = self.W_O(attn_output)
        
        return output, weights
```

### 5.2 使用示例

```python
# 创建多头注意力层
mha = MultiHeadAttention(d_model=512, num_heads=8)

# 输入
batch_size, seq_len = 2, 10
x = torch.randn(batch_size, seq_len, 512)

# 自注意力：Q = K = V = x
output, attn_weights = mha(x, x, x)

print(f"输入形状: {x.shape}")           # [2, 10, 512]
print(f"输出形状: {output.shape}")      # [2, 10, 512]
print(f"注意力权重: {attn_weights.shape}") # [2, 8, 10, 10]
```

## 6. 多头在学什么？

### 6.1 可视化不同头的注意力

研究发现，不同头确实在关注不同类型的关系：

```
头1（语法）: The → cat, cat → sat, sat → on
头2（指代）: the → cat, the → mat
头3（位置）: 相邻词之间权重高
头4（语义）: cat ↔ mat（相似物体）
```

### 6.2 注意力头的重要性

并不是所有头都同样重要：
- 有些头专注于语法模式
- 有些头专注于位置模式
- 有些头可能是冗余的（可以剪枝）

## 7. 参数效率分析

### 计算量对比

```
单头注意力：O(n² × d)
多头注意力：O(n² × d)  总量不变！

因为：h 个头 × (n² × d/h) = n² × d
```

### 参数量

```
单头: 3 × d × d (Q/K/V) + d × d (O) = 4d²
多头: 同样是 4d²

区别在于：同样的参数，学到了更丰富的模式
```

## 8. 实际应用中的选择

### 常见配置

| 模型 | d_model | num_heads | d_k |
|------|---------|-----------|-----|
| Transformer Base | 512 | 8 | 64 |
| BERT Base | 768 | 12 | 64 |
| GPT-2 Small | 768 | 12 | 64 |
| GPT-3 175B | 12288 | 96 | 128 |

### 经验法则

- d_k 通常设为 64（平衡性能和效率）
- 头数通常是 d_model / 64
- 更大的模型用更多的头

## 9. 常见问题

### Q1: 为什么 d_model 必须能被 num_heads 整除？

因为要均匀地拆分维度。512 / 8 = 64，如果不能整除就会有维度不一致。

### Q2: 多头一定比单头好吗？

通常是的，但也有例外：
- 太多头会增加计算开销
- 有些任务简单，单头可能就够了
- 可以通过实验确定最佳头数

### Q3: 不同头之间有信息交互吗？

在注意力计算时没有，但拼接后通过 W_O 矩阵混合了信息。

### Q4: 可以对不同头用不同的 d_k 吗？

理论上可以，但实践中不这样做，因为：
- 实现复杂
- GPU 不友好
- 没有明显收益

---

## 10. 核心要点总结

```
多头注意力 = 多个视角 × 并行计算

拆分：大维度 → 多个小维度
计算：每个头独立做注意力
拼接：小维度 → 大维度
投影：混合不同头的信息

优势：同样的参数，学得更丰富
成本：没有额外计算量（总量不变）
```
