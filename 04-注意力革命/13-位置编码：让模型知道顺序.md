# 13-位置编码：让模型知道顺序

上一章我们完成了多头注意力——它能同时从多个角度捕捉词与词之间的关系。但还有一个**致命问题**没解决：

**注意力机制对顺序不敏感。**

```
"我爱你" 和 "你爱我"
在注意力眼里，完全一样！
```

因为注意力只计算内容的相似度，词的位置信息完全被忽略。打乱顺序，注意力分数不会变。

**位置编码**就是来解决这个问题的——给每个位置加上一个"地址标签"，让模型知道"我是谁"的同时也知道"我在哪"。

---

## 1. 注意力的顺序盲区

### 1.1 问题本质

考虑这两个句子：
- "猫追老鼠"（主语是猫）
- "老鼠追猫"（主语是老鼠）

词完全一样，只是顺序不同，意思截然相反。但注意力计算时：

```
注意力矩阵（无论是哪种顺序）：
      猫    追    老鼠
猫  [0.5   0.3   0.2]
追  [0.25  0.5   0.25]
老鼠[0.2   0.3   0.5]
```

**注意力分数完全相同**——模型无法区分谁追谁。

### 1.2 为什么RNN没有这个问题？

回顾RNN/LSTM：
- 词是一个接一个输入的
- 每个时刻的隐藏状态天然包含了"前面有哪些词"的信息
- 顺序内建在模型结构中

但注意力是**并行计算**所有位置的关系，天生就是"平面"的，没有先后概念。

### 1.3 解决方案

给每个位置添加一个**唯一标识**，让模型能区分：
```
位置0的"猫" ≠ 位置2的"猫"
```

这就是**位置编码（Position Embedding）**。

---

## 2. 方案一：正弦位置编码（Sinusoidal）

### 2.1 原始 Transformer 的选择

论文 "Attention Is All You Need" 提出的方法，不用学习，直接用公式生成。

### 2.2 公式

对于位置 pos 和维度 i：

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

其中：
- pos：位置索引（0, 1, 2, ...）
- i：维度索引
- d_model：模型维度
- 偶数维用 sin，奇数维用 cos

### 2.3 直觉理解

**不同频率的正弦波**：

```
维度0-1：频率很高（快速变化）→ 区分相邻位置
维度2-3：频率中等
维度4-5：频率较低
...
最高维：频率很低（缓慢变化）→ 区分远距离位置
```

类比**二进制编码**：
- 二进制：每一位不同频率（2^0, 2^1, 2^2...）
- 位置编码：正弦波不同频率（sin, sin(10x), sin(100x)...）

### 2.4 代码实现

```python
import torch
import math

def sinusoidal_encoding(max_len, d_model):
    """生成正弦位置编码"""
    pe = torch.zeros(max_len, d_model)
    position = torch.arange(0, max_len).unsqueeze(1).float()
    
    # 计算频率的分母
    div_term = torch.exp(
        torch.arange(0, d_model, 2).float() * 
        -(math.log(10000.0) / d_model)
    )
    
    # 偶数维用 sin，奇数维用 cos
    pe[:, 0::2] = torch.sin(position * div_term)
    pe[:, 1::2] = torch.cos(position * div_term)
    
    return pe

# 使用
pe = sinusoidal_encoding(max_len=512, d_model=512)
# pe[i] 就是位置 i 的编码向量

# 在模型中使用
x = token_embedding + pe[:seq_len]  # 直接加到词嵌入上
```

### 2.5 优点

1. **无需学习**：公式生成，不增加参数
2. **外推性好**：公式可以生成任意位置的编码
3. **相对位置可学**：PE(pos+k) 可以用 PE(pos) 的线性变换表示

## 3. 方案二：RoPE（旋转位置编码）

### 3.1 动机

正弦编码直接加在嵌入上，但注意力计算时位置信息可能被稀释。

**RoPE 的思路**：不在嵌入上加，而是在 Q/K 计算时旋转。

### 3.2 核心思想

用**旋转矩阵**编码位置：
```
Q' = R(pos) × Q
K' = R(pos) × K

其中 R(pos) 是旋转矩阵
```

这样 Q·K 的点积自然包含了相对位置信息。

### 3.3 公式

对于二维向量 (x₁, x₂)，旋转角度 θ：

```
旋转后：
x₁' = x₁ cos(θ) - x₂ sin(θ)
x₂' = x₁ sin(θ) + x₂ cos(θ)

矩阵形式：
[x₁']   [cos(θ)  -sin(θ)] [x₁]
[x₂'] = [sin(θ)   cos(θ)] [x₂]
```

对于位置 pos，角度 θ_pos = pos × θ_base，其中 θ_base 是基础频率。

### 3.4 实现细节

```python
def rotate_half(x):
    """将向量的后半部分取反并移到前面"""
    x1, x2 = x.chunk(2, dim=-1)
    return torch.cat([-x2, x1], dim=-1)

def apply_rotary_pos_emb(q, k, cos, sin):
    """应用旋转位置编码"""
    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)
    return q_embed, k_embed

def precompute_freqs_cis(dim, max_len, theta=10000.0):
    """预计算频率"""
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[:dim//2].float() / dim))
    t = torch.arange(max_len)
    freqs = torch.outer(t, freqs)  # [max_len, dim//2]
    freqs_cos = torch.cos(freqs)   # [max_len, dim//2]
    freqs_sin = torch.sin(freqs)   # [max_len, dim//2]
    return freqs_cos, freqs_sin

# 使用
freqs_cos, freqs_sin = precompute_freqs_cis(d_k, max_len)
q_rotated, k_rotated = apply_rotary_pos_emb(
    q, k,
    freqs_cos[:seq_len],
    freqs_sin[:seq_len]
)
# 然后用 q_rotated, k_rotated 计算注意力
```

### 3.5 为什么 RoPE 有效？

关键性质：
```
Q(pos1) · K(pos2) = Q^T × R(pos1)^T × R(pos2) × K
                  = Q^T × R(pos2 - pos1) × K
```

点积结果只依赖于**相对距离** (pos2 - pos1)！

### 3.6 应用

- **LLaMA**：使用 RoPE
- **Mistral**：使用 RoPE
- **Qwen**：使用 RoPE
- 目前最流行的方案之一

## 4. 方案三：ALiBi（Attention with Linear Biases）

### 4.1 动机

RoPE 需要对 Q/K 做变换，ALiBi 更简单：直接在注意力分数上加偏置。

### 4.2 核心思想

给注意力分数加上一个**与距离成正比的惩罚**：
```
Attention_score = QK^T / √d_k + linear_bias

其中 linear_bias = -m × |i - j|
m 是每个头不同的斜率
|i - j| 是位置距离
```

**直觉**：距离越远，惩罚越大（注意力分数越低）。

### 4.3 为什么用不同的斜率？

不同的注意力头用不同的斜率 m：

```
头1: m = 1/2   → 惩罚很重，关注近距离
头2: m = 1/4   → 惩罚中等
头3: m = 1/8   → 惩罚较轻
头4: m = 1/16  → 几乎不惩罚，看全局
```

这样不同头能关注不同距离范围。

### 4.4 代码实现

```python
def get_alibi_bias(num_heads, seq_len):
    """生成 ALiBi 偏置矩阵"""
    # 每个头的斜率（几何序列）
    slopes = torch.pow(2, -8/num_heads * torch.arange(1, num_heads + 1))
    
    # 距离矩阵
    positions = torch.arange(seq_len)
    distance = positions.unsqueeze(0) - positions.unsqueeze(1)  # [seq, seq]
    distance = distance.abs()  # 取绝对值
    
    # 偏置 = -斜率 × 距离
    bias = -slopes.view(num_heads, 1, 1) * distance.unsqueeze(0)
    # bias: [num_heads, seq_len, seq_len]
    
    return bias

# 使用
alibi_bias = get_alibi_bias(num_heads=8, seq_len=seq_len)

# 在注意力计算中
scores = QK^T / √d_k + alibi_bias  # 直接加到分数上
weights = softmax(scores)
```

### 4.5 优点

1. **最简单**：只加一个偏置矩阵
2. **无需修改 Q/K**：不影响其他部分
3. **外推性好**：线性偏置天然支持长序列
4. **零额外参数**：斜率是固定的

### 4.6 应用

- **BLOOM**：使用 ALiBi
- **MPT**：使用 ALiBi

## 5. 三种方案对比

| 特性 | 正弦编码 | RoPE | ALiBi |
|------|----------|------|-------|
| **实现方式** | 加到嵌入上 | 旋转 Q/K | 加到分数上 |
| **参数量** | 0 | 0 | 0 |
| **相对位置** | 间接表达 | 直接编码 | 直接编码 |
| **外推能力** | 好 | 需要改进 | 最好 |
| **复杂度** | 低 | 中 | 最低 |
| **流行度** | 原始方案 | 最流行 | 流行 |

### 选择建议

```
- 入门学习：正弦编码（最直观）
- 新项目：RoPE（效果好，社区支持多）
- 长序列：ALiBi（外推性最好）
```

## 6. 位置编码的加入方式

### 方式一：加法（Add）

最常见，直接加到词嵌入上：
```python
x = token_embedding + position_embedding
```

### 方式二：拼接（Concat）

```python
x = torch.cat([token_embedding, position_embedding], dim=-1)
```

### 方式三：注入（RoPE/ALiBi）

不在嵌入层加入，而在注意力计算时加入。

## 8. 常见问题

### Q1: 位置编码加在哪？

在最开始，和词嵌入相加：
```python
x = word_embedding(input_ids) + position_embedding
```

或者在注意力计算时（RoPE/ALiBi）。

### Q2: 位置编码会不会被后面的层"冲掉"？

实验证明不会。位置信息在各层中保持得很好。

### Q3: 为什么不用可学习的位置编码？

可以，BERT/GPT 都用可学习的：
- 优点：简单
- 缺点：限制序列长度，不能外推

正弦/RoPE/ALiBi 能处理训练时没见过的长度。

### Q4: RoPE 的 base frequency 是什么意思？

控制旋转的"粒度"：
- θ_base = 10000：原始 Transformer 的值
- θ_base = 1000000：LLaMA 2 的值（支持更长序列）

## 9. 代码总结

```python
import torch
import torch.nn as nn

class PositionalEncoding(nn.Module):
    """正弦位置编码"""
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(max_len).unsqueeze(1).float()
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * 
            -(math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        self.register_buffer('pe', pe)  # 不参与训练
    
    def forward(self, x):
        return x + self.pe[:x.size(1)]

# 使用
pos_enc = PositionalEncoding(d_model=512)
x = token_embedding(input_ids)  # [batch, seq, 512]
x = pos_enc(x)  # 加上位置编码
```
