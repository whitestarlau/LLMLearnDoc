# 15-语言模型

我们已经学习了 Transformer 的完整架构。现在要问：这套架构是用来做什么的？

答案是：**让机器学会人类的语言规律**。

这就是**语言模型**（Language Model, LM）——NLP 领域最核心、最基础的任务。从搜索引擎的自动补全，到 GPT-4 写代码，背后都是语言模型。

有趣的是，语言模型这个概念比深度学习早得多。它经历了从统计方法到神经网络，再到 Transformer 的漫长演进。理解这段历史，才能真正理解现代大模型的本质。

---

## 1. 什么是语言模型？

### 1.1 核心定义

语言模型回答一个问题：**这个句子出现的概率是多少？**

```
P("我喜欢吃苹果") = ?
P("我苹果喜欢吃") = ?
```

好的语言模型会给出：
```
P("我喜欢吃苹果") > P("我苹果喜欢吃")
```

因为前者更符合人类的语言习惯。

### 1.2 链式法则

一个句子的概率可以拆分成条件概率的连乘：

```
P("我 喜欢 吃 苹果") = 
  P(我) × P(喜欢|我) × P(吃|我,喜欢) × P(苹果|我,喜欢,吃)
```

用数学表达：

$$
P(x_1, x_2, ..., x_n) = \prod_{i=1}^{n} P(x_i | x_1, ..., x_{i-1})
$$

**这就是语言模型的本质**：学会计算每一个条件概率 $P(x_i | x_1, ..., x_{i-1})$。

### 1.3 一个模型，三种用法

基于这个核心能力，语言模型可以做三件事：

| 用法 | 做什么 | 示例 |
|------|--------|------|
| **评分** | 给句子打分 | 判断"我喜欢苹果"比"我苹果喜欢"更通顺 |
| **预测** | 猜下一个词 | 看到"今天天气"，预测下一个是"很好" |
| **生成** | 写完整文本 | 从"今天"开始，一个词一个词生成一段话 |

**关键洞察**：这三种能力都来自于同一个核心——预测下一个词的概率。预测能力越强，评分和生成就越好。

## 2. 语言模型的演进史

### 2.1 统计语言模型（1990s-2000s）

**核心思想**：统计词在语料库中出现的频率。

**N-gram 模型**：
- 假设一个词只依赖于前面 N-1 个词
- N=2（Bigram）：$P(x_i | x_{i-1})$
- N=3（Trigram）：$P(x_i | x_{i-1}, x_{i-2})$

```python
# 从语料库统计
P("喜欢" | "我") = count("我喜欢") / count("我")
P("吃" | "喜欢") = count("喜欢吃") / count("喜欢")
```

**局限**：
- 数据稀疏：很多组合在语料库中从未出现
- 上下文短：只能看前2-3个词
- 没有语义理解："猫坐在垫子上"和"狗坐在垫子上"被视为完全不同的模式

### 2.2 神经网络语言模型（2000s-2010s）

**突破**：用神经网络学习词的分布式表示。

**NNLM（Neural Network Language Model, 2003）**：
- Yoshua Bengio 提出
- 用词嵌入（Word Embedding）表示词
- 前馈网络预测下一个词

**RNNLM（2010）**：
- 用 RNN 建模长距离依赖
- 可以记住更长的上下文
- Tomas Mikolov 提出，后来发明了 Word2Vec

**进步**：
- 能处理未见过的新组合
- 语义相似的词有相似的嵌入
- 但仍然受限于 RNN 的序列计算

### 2.3 大规模语言模型（2018-至今）

**Transformer 的革命**：
- 2017 年 Transformer 论文
- 2018 年 GPT-1 和 BERT
- 开启了预训练 + 微调的时代

**核心变化**：
- 模型规模：百万 → 十亿 → 万亿参数
- 数据规模：百万 → 千亿 token
- 从"学习语言规律"变成"学习世界知识"

**关键人物与时间线**：

| 时间 | 事件 | 意义 |
|------|------|------|
| 2003 | Bengio 的 NNLM | 神经网络语言模型开端 |
| 2010 | RNNLM | 循环神经网络建模序列 |
| 2013 | Word2Vec | 词嵌入普及 |
| 2017 | Transformer | 注意力机制取代 RNN |
| 2018 | GPT-1 / BERT | 预训练语言模型时代 |
| 2019 | GPT-2 | 展示涌现能力 |
| 2020 | GPT-3 | 上下文学习（In-context Learning） |
| 2022 | ChatGPT | 对话能力突破 |
| 2023 | GPT-4 | 多模态能力 |

## 3. 现代语言模型的核心机制

### 3.1 自回归（Autoregressive）

**定义**：用过去的自己预测未来的自己。

```
P(x₁, x₂, x₃, x₄) = P(x₁) × P(x₂|x₁) × P(x₃|x₁,x₂) × P(x₄|x₁,x₂,x₃)
```

**训练示例**：

```
文本: "今天天气很好"

训练样本1:
  输入: [今天]
  目标: 天气

训练样本2:
  输入: [今天, 天气]
  目标: 很

训练样本3:
  输入: [今天, 天气, 很]
  目标: 好
```

这就是**Next Token Prediction**——语言模型的核心训练任务。

### 3.2 训练过程

```python
def train_step(model, batch):
    inputs, targets = batch
    # inputs: [batch, seq_len]
    # targets: [batch, seq_len]
    
    logits = model(inputs)  # [batch, seq_len, vocab_size]
    
    # 对每个位置预测下一个词
    loss = F.cross_entropy(
        logits.reshape(-1, vocab_size),
        targets.reshape(-1)
    )
    
    return loss
```

**关键点**：
- 输入和目标错位一位（inputs = targets 左移）
- 用 Teacher Forcing：训练时用真实词作为输入
- 损失函数：交叉熵

### 3.3 生成过程

训练好的模型可以生成文本：

```python
def generate(model, prompt, max_tokens=100, temperature=1.0):
    tokens = tokenize(prompt)
    
    for _ in range(max_tokens):
        # 前向传播
        logits = model(tokens)
        
        # 取最后一个位置的预测
        next_logits = logits[:, -1, :] / temperature
        
        # 采样下一个词
        probs = F.softmax(next_logits, dim=-1)
        next_token = torch.multinomial(probs, num_samples=1)
        
        # 拼接
        tokens = torch.cat([tokens, next_token], dim=1)
        
        if next_token.item() == EOS_TOKEN:
            break
    
    return detokenize(tokens)
```

**核心循环**：
1. 把已生成的词输入模型
2. 模型预测下一个词的概率分布
3. 从分布中采样一个词
4. 重复直到生成结束符或达到最大长度

## 4. 语言模型的类型

现代基于 Transformer 的语言模型主要有三种架构：

### 4.1 自回归模型（AR）

**代表**：GPT 系列

**特点**：
- 单向注意力（只能看左边的词）
- 自左向右生成
- 适合：文本生成、对话、代码补全

```
输入: "今天天气"
输出: "很好，适合出门"
```

### 4.2 自编码模型（AE）

**代表**：BERT

**特点**：
- 双向注意力（能看到左右两边的词）
- 随机遮盖（[MASK]）预测
- 适合：文本理解、分类、抽取

```
输入: "今天 [MASK] 很好"
输出: "天气"
```

### 4.3 编码器-解码器模型

**代表**：T5、BART

**特点**：
- 编码器双向理解输入
- 解码器自回归生成输出
- 适合：翻译、摘要、问答

```
输入（编码器）: "今天天气很好"
输出（解码器）: "The weather is good today"
```

### 4.4 对比

| 类型 | 代表 | 注意力方向 | 典型任务 |
|------|------|-----------|---------|
| 自回归 | GPT | 单向（左→右） | 生成 |
| 自编码 | BERT | 双向 | 理解 |
| 编解码 | T5 | 编码器双向 + 解码器单向 | 翻译、摘要 |

## 5. 评估指标：困惑度（Perplexity）

### 5.1 什么是困惑度？

**困惑度**（PPL）衡量语言模型"有多困惑"。

- PPL = 1：完全不困惑，完美预测
- PPL = 100：相当于面对100个等概率选择
- PPL = 10000：基本随机

**直觉**：困惑度 ≈ 模型每次预测时面对的有效选择数

### 5.2 数学定义

$$
PPL = \exp\left(-\frac{1}{N} \sum_{i=1}^{N} \log P(x_i | x_1, ..., x_{i-1})\right)
$$

等价于交叉熵损失的指数：

```python
ppl = torch.exp(cross_entropy_loss)
```

### 5.3 典型模型的困惑度

| 模型 | 年份 | 规模 | PPL (WikiText-103) |
|------|------|------|-------------------|
| RNNLM | 2010 | 很小 | ~100 |
| GPT-1 | 2018 | 117M | ~30 |
| GPT-2 | 2019 | 1.5B | ~20 |
| GPT-3 | 2020 | 175B | ~10 |

## 6. 生成策略

### 6.1 贪心搜索（Greedy Search）

每次选概率最高的词：

```python
def greedy_search(model, prompt, max_tokens=100):
    tokens = tokenize(prompt)
    
    for _ in range(max_tokens):
        logits = model(tokens)
        next_token = logits[:, -1].argmax(keepdim=True)
        tokens = torch.cat([tokens, next_token], dim=1)
    
    return tokens
```

**缺点**：容易陷入局部最优，重复生成相同内容。

### 6.2 束搜索（Beam Search）

同时保持 k 个最优候选：

```python
def beam_search(model, prompt, beam_size=5, max_tokens=100):
    tokens = tokenize(prompt)
    candidates = [(tokens, 0)]  # (序列, 对数概率)
    
    for _ in range(max_tokens):
        new_candidates = []
        
        for seq, score in candidates:
            logits = model(seq)
            log_probs = F.log_softmax(logits[:, -1], dim=-1)
            top_probs, top_tokens = log_probs.topk(beam_size)
            
            for prob, token in zip(top_probs[0], top_tokens[0]):
                new_seq = torch.cat([seq, token.unsqueeze(0).unsqueeze(0)], dim=1)
                new_candidates.append((new_seq, score + prob.item()))
        
        candidates = sorted(new_candidates, key=lambda x: x[1], reverse=True)[:beam_size]
    
    return candidates[0][0]
```

**适用**：翻译、摘要等确定性任务。

### 6.3 采样方法

**温度采样**：

```python
def sample_with_temperature(logits, temperature=1.0):
    logits = logits / temperature
    probs = F.softmax(logits, dim=-1)
    return torch.multinomial(probs, num_samples=1)
```

- temperature → 0：接近贪心
- temperature = 1：正常采样
- temperature > 1：更随机、更有创意

**Top-k 采样**：只从概率最高的 k 个词中采样

**Top-p（Nucleus）采样**：从累积概率达到 p 的最小集合中采样

## 7. 从语言模型到大模型

### 7.1  scaling 的威力

2018-2023 年的一个重要发现：**模型越大，能力越强**。

| 模型 | 参数量 | 数据量 | 新能力 |
|------|--------|--------|--------|
| GPT-1 | 117M | 4GB | 基础生成 |
| GPT-2 | 1.5B | 40GB | 零样本能力 |
| GPT-3 | 175B | 570GB | 上下文学习 |
| GPT-4 | 估计1T+ | 未知 | 推理、多模态 |

### 7.2 涌现能力（Emergent Abilities）

当模型规模超过某个阈值，会突然表现出小模型没有的能力：

- **上下文学习**（In-context Learning）：给几个例子就能学会新任务
- **思维链**（Chain-of-Thought）：能一步步推理
- **指令遵循**（Instruction Following）：按人类指令行动

### 7.3 现代大模型的工作流程

```
1. 预训练（Pre-training）
   ↓ 在大量文本上训练，学习语言规律和世界知识
   
2. 有监督微调（SFT）
   ↓ 在指令-回答对上训练，学会对话格式
   
3. 人类反馈强化学习（RLHF）
   ↓ 让人类评价回答质量，训练奖励模型，优化策略
   
4. 部署使用
   → 接收用户输入，生成回答
```

## 8. 常见问题

### Q1: 最开始的语言模型是 GPT 吗？

不是。语言模型的概念可以追溯到 1980-1990 年代的统计语言模型（N-gram）。GPT 是 2018 年才出现的神经网络语言模型。

### Q2: 为什么 GPT 用解码器而不是编码器？

解码器的**因果掩码**（Causal Mask）天然适合自回归生成——每次只能看到前面的词，不能"偷看"后面的词。编码器的双向注意力更适合理解任务（如 BERT）。

### Q3: 语言模型真的"理解"语言吗？

这是哲学问题。从功能上，现代大模型能：
- 回答问题
- 推理
- 写代码
- 翻译

但它们是否有人类一样的"理解"，目前仍有争议。

### Q4: 为什么大模型能回答训练数据中没有的问题？

因为语言模型学到的不是"记忆"，而是**规律**。就像你学会了数学规则，就能解没见过的数学题。

## 9. 核心要点总结

```
语言模型的本质：
  P(句子) = P(词1) × P(词2|词1) × P(词3|词1,词2) × ...

历史演进：
  统计语言模型（N-gram） → 神经网络语言模型（RNN） → 大规模语言模型（Transformer）

核心任务：
  Next Token Prediction（预测下一个词）

三种架构：
  - 自回归（GPT）：单向生成
  - 自编码（BERT）：双向理解
  - 编解码（T5）：翻译、摘要

评估指标：
  困惑度（Perplexity）越低越好

生成策略：
  - 贪心：每次选最好的
  - 束搜索：保持多个候选
  - 采样：温度、Top-k、Top-p
```
