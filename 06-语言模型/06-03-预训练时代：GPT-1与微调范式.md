# 06-03 预训练时代：GPT-1与微调范式

我们已经了解了 GPT 的架构——一个只用解码器的 Transformer。但 GPT 真正的革命性创新，不在于架构本身，而在于**训练范式**。

2018年，OpenAI 发布 GPT-1，提出了"**预训练 + 微调**"的两阶段训练方法。这一范式彻底改变了 NLP 领域，成为后来 BERT、GPT-2/3/4 乃至所有现代大模型的基础。

在这一章，我们将深入理解：为什么需要预训练？预训练学到了什么？微调又是如何让模型适应具体任务的？

---

## 1. GPT 之前的 NLP 范式

### 1.1 传统方法的痛点

在 GPT-1 之前，NLP 任务通常采用"**从零训练**"的方法：

```
情感分类任务：
1. 收集标注数据（几千到几万条）
2. 人工设计特征（TF-IDF、n-gram、情感词典）
3. 训练分类器（SVM、朴素贝叶斯、浅层神经网络）
4. 在测试集上评估

命名实体识别任务：
1. 收集标注数据（标注人名、地名、组织名）
2. 人工设计特征（词性、词边界、词典匹配）
3. 训练序列标注模型（CRF、HMM）
4. 在测试集上评估

机器翻译任务：
1. 收集平行语料（人工翻译的句子对）
2. 训练统计机器翻译模型
3. 调参优化
```

**三大痛点**：

1. **标注数据稀缺**
   - 高质量标注成本高昂
   - 每个任务需要独立的标注数据
   - 小数据集容易过拟合

2. **特征工程依赖人工**
   - 需要领域专家设计特征
   - 不同任务需要不同的特征
   - 无法自动学习语义相似性

3. **知识无法复用**
   - 每个任务独立训练模型
   - 学到的语言知识无法迁移
   - 计算资源浪费严重

### 1.2 一个具体例子

**情感分类的传统方法**：

```python
# 传统方法：特征工程 + 分类器
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.svm import SVC

# 1. 人工设计特征：TF-IDF
vectorizer = TfidfVectorizer(
    max_features=10000,      # 限制特征数量
    ngram_range=(1, 2),      # 使用 unigram 和 bigram
    stop_words='english'     # 去除停用词
)

# 2. 训练 SVM 分类器
classifier = SVC(kernel='linear')
pipeline = Pipeline([
    ('tfidf', vectorizer),
    ('clf', classifier)
])

# 3. 需要大量标注数据训练
pipeline.fit(train_texts, train_labels)

# 问题：
# - "好吃"和"美味"被视为完全不同的特征
# - 无法理解语义相似性
# - 换个任务（如新闻分类）要重新开始
```

### 1.3 传统 vs GPT 范式对比

```
传统范式（每个任务独立）：
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 情感分类模型  │    │ 新闻分类模型  │    │ 问答模型      │
│ (TF-IDF+SVM) │    │ (TF-IDF+SVM) │    │ (规则+统计)  │
└──────────────┘    └──────────────┘    └──────────────┘
      ↓                   ↓                   ↓
  10K 标注            15K 标注            50K 标注
  3小时训练           4小时训练           2天训练
      ↓                   ↓                   ↓
   无法复用            无法复用            无法复用
```

```
GPT 范式（预训练 + 微调）：
┌─────────────────────────────────────────────────────┐
│              预训练 GPT 模型（只需一次）              │
│         在海量无标注文本上学习通用语言表示            │
│                   117M 参数                         │
│              训练时间：几天到几周                     │
└─────────────────────────────────────────────────────┘
                         ↓
         可复用到所有下游任务
                         ↓
    ┌──────────┬──────────┬──────────┐
    ↓          ↓          ↓          ↓
┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐
│情感分类│  │新闻分类│  │ 问答   │  │翻译    │
│+分类头 │  │+分类头 │  │+生成头 │  │+生成头 │
└───────┘  └───────┘  └───────┘  └───────┘
   1K标注     1K标注    5K标注     10K标注
  30分钟     30分钟    1小时      2小时
```

**核心优势**：预训练模型学到通用语言能力，只需少量标注数据微调即可适应具体任务。

## 2. GPT-1 的两阶段范式

### 2.1 论文背景

**论文标题**："Improving Language Understanding by Generative Pre-Training"

**作者**：Alec Radford, Karthik Narasimhan, Tim Salimans, Ilya Sutskever (OpenAI)

**时间**：2018年6月

**核心思想**：先用无监督预训练学习通用语言表示，再用有监督微调适应具体任务。

### 2.2 为什么是无监督预训练？

**关键洞察**：互联网上有海量的无标注文本，蕴含丰富的语言知识和世界常识。

```
可用的数据：
- BookCorpus：8000本书，约5GB文本
- Wikipedia：数百万篇文章
- 互联网网页：数以亿计

标注数据（昂贵）：
- 情感标注：几千到几万条
- 翻译语料：几十万对（需人工翻译）
- 问答数据：几万对
```

**预训练的优势**：
1. **数据无限**：不需要人工标注
2. **知识通用**：学到的表示可迁移
3. **表示丰富**：多层次的语义理解
4. **成本低廉**：只需一次训练

### 2.3 阶段一：无监督预训练

**目标**：从海量文本中学习预测下一个词的能力。

```
标准语言建模目标：
L₁(𝒰) = Σᵢ log P(uᵢ | uᵢ₋ₖ, ..., uᵢ₋₁; Θ)

其中：
- 𝒰 = {u₁, u₂, ..., uₙ} 是无标注语料
- k 是上下文窗口大小
- Θ 是模型参数
```

**训练过程**：

```python
# 预训练伪代码
for batch in unlabeled_data:
    # 输入和目标错位一位
    inputs = batch[:, :-1]    # [今天, 天气, 很]
    targets = batch[:, 1:]    # [天气, 很, 好]
    
    # 前向传播
    logits = model(inputs)    # [batch, seq_len, vocab_size]
    
    # 计算交叉熵损失
    loss = F.cross_entropy(
        logits.reshape(-1, vocab_size),
        targets.reshape(-1)
    )
    
    # 反向传播
    loss.backward()
    optimizer.step()
```

**预训练数据**：BookCorpus
- 约8000本未出版的书籍
- 约5GB文本
- 包含各种类型的文本：小说、传记、科普等

### 2.4 预训练学到了什么？

**层次化的语言知识**：

```
预训练学到的知识层次：

┌─────────────────────────────────────────┐
│  表层：词汇、语法、句法结构              │
│  示例：主谓宾结构、时态、单复数          │
├─────────────────────────────────────────┤
│  中层：语义、指代、逻辑关系              │
│  示例："苹果"可以指水果或公司            │
├─────────────────────────────────────────┤
│  深层：常识、世界知识、推理模式          │
│  示例：天气好时人们喜欢户外活动          │
├─────────────────────────────────────────┤
│  任务层：具体任务的决策边界              │
│  示例：分类、问答、翻译的特定模式        │
└─────────────────────────────────────────┘
        ↑
    预训练学到           微调学习
```

**为什么"预测下一个词"能学到这么多？**

```python
# 预测下一个词需要理解：

input = "今天天气很好，我决定去"

# 1. 语法知识
#    "决定去"后面通常接地点或活动

# 2. 常识知识
#    天气好时人们喜欢户外活动

# 3. 世界知识
#    常见的户外活动：公园、海边、爬山...

# 4. 推理能力
#    根据上下文推断最可能的续写

# 预测下一个词 = 压缩所有这些知识
```

### 2.5 阶段二：有监督微调

**目标**：将通用语言能力适配到具体任务。

```
微调目标：
L₂(𝒞) = Σᵢ log P(yᵢ | x₁, ..., xₙ; Θ) + λ L₁(𝒞)

其中：
- 𝒞 是有标注数据集
- (x, y) 是输入-输出对
- λ 是预训练损失的权重（可选）
```

**微调过程**：

```python
# 微调伪代码
# 添加任务特定的输出层
classifier = nn.Linear(d_model, num_classes)

for batch in labeled_data:
    inputs, labels = batch
    
    # 提取特征（冻结或微调底层）
    features = model.extract_features(inputs)
    
    # 分类
    logits = classifier(features[:, -1])  # 取最后一个位置的表示
    
    # 任务损失
    loss = F.cross_entropy(logits, labels)
    
    loss.backward()
    optimizer.step()
```

### 2.6 任务适配：如何把各种任务变成文本生成

GPT-1 的创新之一：**统一任务形式**。

不同任务通过特定的输入格式转换成文本生成问题：

**分类任务（如情感分析）**：
```
输入: [Start] 这部电影真的很精彩！ [Extract]
输出: Positive

输入: [Start] 太失望了，完全不值这个票价。 [Extract]
输出: Negative
```

**文本蕴含（Entailment）**：
```
输入: [Start] 前提文本 [Delim] 假设文本 [Extract]
输出: Entailment / Contradiction / Neutral
```

**语义相似度**：
```
输入: [Start] 句子A [Delim] 句子B [Extract]
输出: Similar / Dissimilar
```

**问答**：
```
输入: [Start] 上下文 [Delim] 问题 [Extract]
输出: 答案
```

**特殊符号**：
- `[Start]`：输入开始
- `[Delim]`：分隔不同部分
- `[Extract]`：提取特征的位置

## 3. GPT-1 的效果

### 3.1 实验结果

GPT-1 在多个 NLP 任务上取得了当时的最佳性能（SOTA）：

| 任务类型 | 数据集 | GPT-1 | 之前最佳 | 提升 |
|---------|--------|-------|---------|------|
| 自然语言推理 | SNLI | 89.9% | 86.7% | +3.2% |
| 问答 | SQuAD | 75.0 F1 | 71.2 F1 | +3.8 |
| 语义相似度 | STS-B | 82.0 | 79.7 | +2.3 |
| 文本分类 | AGNews | 95.2% | 93.3% | +1.9% |

### 3.2 关键发现

**1. 预训练的价值**

对比实验（从头训练 vs 预训练+微调）：

| 训练方式 | SNLI准确率 | 训练数据 |
|---------|-----------|---------|
| 从头训练 | 79.2% | 100% |
| 预训练+微调 | 89.9% | 100% |
| 预训练+微调 | 88.4% | 10% |

**结论**：即使只用10%的标注数据，预训练模型的效果也远超从头训练。

**2. 层数的影响**

| 层数 | 参数量 | SNLI准确率 |
|------|--------|-----------|
| 4 | 20M | 83.2% |
| 8 | 60M | 86.8% |
| 12 | 117M | 89.9% |

更深的模型效果更好。

**3. 预训练数据量的影响**

| 数据量 | SNLI准确率 |
|--------|-----------|
| 1% | 82.1% |
| 10% | 86.3% |
| 100% | 89.9% |

更多预训练数据带来更好的效果。

## 4. 微调策略详解

### 4.1 输出层设计

根据不同任务类型，添加不同的输出层：

**分类任务**：
```python
class Classifier(nn.Module):
    def __init__(self, gpt, num_classes):
        super().__init__()
        self.gpt = gpt
        self.classifier = nn.Linear(gpt.d_model, num_classes)
    
    def forward(self, input_ids):
        # GPT 提取特征
        features = self.gpt(input_ids)  # [B, T, d_model]
        
        # 取最后一个位置的特征（或[Extract]位置）
        pooled = features[:, -1]  # [B, d_model]
        
        # 分类
        logits = self.classifier(pooled)  # [B, num_classes]
        return logits
```

**序列标注任务**（如NER）：
```python
class TokenClassifier(nn.Module):
    def __init__(self, gpt, num_labels):
        super().__init__()
        self.gpt = gpt
        self.classifier = nn.Linear(gpt.d_model, num_labels)
    
    def forward(self, input_ids):
        features = self.gpt(input_ids)  # [B, T, d_model]
        logits = self.classifier(features)  # [B, T, num_labels]
        return logits
```

### 4.2 微调技巧

**学习率设置**：
```python
# 底层使用较小学习率，顶层使用较大学习率
optimizer = AdamW([
    {'params': model.gpt.blocks[:6].parameters(), 'lr': 1e-5},   # 底层
    {'params': model.gpt.blocks[6:].parameters(), 'lr': 5e-5},   # 上层
    {'params': model.classifier.parameters(), 'lr': 1e-4}        # 分类头
])
```

**层数解冻策略**：

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| 全微调 | 训练所有参数 | 数据充足 |
| 顶层微调 | 冻结底层，只调顶层 | 数据较少 |
| 逐层解冻 | 先训练顶层，再逐步解冻 | 中等数据 |

**早停（Early Stopping）**：
```python
best_val_loss = float('inf')
patience = 3
counter = 0

for epoch in range(max_epochs):
    train(model, train_loader)
    val_loss = evaluate(model, val_loader)
    
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        save_checkpoint(model)
        counter = 0
    else:
        counter += 1
        if counter >= patience:
            print("Early stopping!")
            break
```

## 5. 预训练+微调范式的意义

### 5.1 对 NLP 领域的影响

**1. 范式转变**

从"任务特定模型"转向"通用预训练模型"。

**2. 降低门槛**

- 不需要大量标注数据
- 不需要复杂的特征工程
- 预训练模型可直接使用

**3. 统一架构**

所有任务都可以用 GPT 架构解决。

### 5.2 后续发展

**2018年**：GPT-1 + BERT（双向预训练）

**2019年**：GPT-2（零样本学习）

**2020年**：GPT-3（少样本学习、In-Context Learning）

**2022年**：ChatGPT（RLHF对齐）

### 5.3 核心思想总结

```
预训练 + 微调 = 通用语言能力 + 任务适配

预训练：
- 数据：海量无标注文本
- 目标：预测下一个词
- 结果：通用语言表示

微调：
- 数据：少量标注数据
- 目标：任务特定输出
- 结果：任务专用模型

优势：
- 数据效率高
- 知识可迁移
- 训练成本低
```

## 6. 小结

**GPT-1 的核心贡献**：

1. **提出了"预训练 + 微调"范式**
   - 无监督预训练学习通用表示
   - 有监督微调适应具体任务

2. **验证了预训练的价值**
   - 即使少量标注数据也能取得好效果
   - 远超从头训练的模型

3. **统一了 NLP 任务形式**
   - 所有任务都变成文本生成
   - 统一的架构解决多种任务

**历史意义**：

GPT-1 开启了 NLP 的预训练时代。这一范式被后来的 BERT、GPT-2/3/4 继承和发展，成为现代大模型的基础。
