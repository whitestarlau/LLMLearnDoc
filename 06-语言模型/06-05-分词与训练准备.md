# 06-05 分词与训练准备

我们已经学习了 GPT 的架构和训练范式。现在，让我们来看看实际训练前的准备工作。

**分词（Tokenization）**是训练语言模型的第一步——把文本转换成模型能理解的数字。这一步看似简单，却对模型性能和效率有着深远影响。

本章将介绍主流的分词算法（BPE、WordPiece、SentencePiece），以及数据清洗、数据集构建等关键准备工作。

---

## 1. 为什么需要分词？

### 1.1 文本到数字的桥梁

模型只能处理数字，不能直接理解文字：

```
文本: "我喜欢学习"
    ↓ 分词
["我", "喜欢", "学习"]
    ↓ 映射到数字
[1234, 5678, 9012]
```

### 好的分词器

- **词表大小适中**：不会太大（稀疏）也不会太小（覆盖不足）
- **常见词完整**："学习"不要拆成"学"+"习"
- **罕见词可拆**："鸔鵿"可以拆成字符
- **多语言支持**：中英文都能处理

---

## 2. 分词方法

### 2.1 词级分词（Word-level）

最简单的方法：

```python
text = "I love learning"
tokens = text.split()
# ["I", "love", "learning"]
```

问题：
- 词表太大（英语有几十万词）
- 未登录词（OOV）无法处理
- 形态变化：run, runs, running 是三个词

### 2.2 字符级分词（Character-level）

把每个字符当作 token：

```python
text = "I love"
tokens = list(text)
# ['I', ' ', 'l', 'o', 'v', 'e']
```

优点：
- 词表极小（几十到几百）
- 没有 OOV 问题

缺点：
- 序列太长
- 学习不到词级别的语义

### 2.3 子词分词（Subword）

结合词级和字符级的优点：

- 常见词保持完整
- 罕见词拆成子词
- 平衡词表大小和序列长度

主流方法：
- **BPE**（Byte Pair Encoding）
- **WordPiece**
- **Unigram**
- **SentencePiece**

## 3. BPE（Byte Pair Encoding）

### 3.1 算法原理

**迭代合并最频繁的相邻字符对**：

```
初始: ["l", "o", "w"], ["l", "o", "w", "e", "r"], ["n", "e", "w", "e", "s", "t"]

统计频率:
(l, o): 2
(o, w): 2
(w, e): 2
...

合并 (l, o):
["lo", "w"], ["lo", "w", "e", "r"], ...

继续合并高频对...
```

### 3.2 训练过程

```python
from collections import Counter

def train_bpe(texts, vocab_size):
    # 1. 初始化：每个字符一个 token
    word_freqs = Counter()
    for text in texts:
        words = text.split()
        for word in words:
            word_freqs[' '.join(list(word)) + ' </w>'] += 1
    
    # 2. 统计字符对频率
    def get_pairs(word_freqs):
        pairs = Counter()
        for word, freq in word_freqs.items():
            symbols = word.split()
            for i in range(len(symbols) - 1):
                pairs[(symbols[i], symbols[i+1])] += freq
        return pairs
    
    # 3. 迭代合并
    vocab = set()
    for word in word_freqs:
        vocab.update(word.split())
    
    merges = []
    while len(vocab) < vocab_size:
        pairs = get_pairs(word_freqs)
        if not pairs:
            break
        
        # 找最高频对
        best_pair = max(pairs, key=pairs.get)
        merges.append(best_pair)
        
        # 合并
        new_token = best_pair[0] + best_pair[1]
        vocab.add(new_token)
        
        # 更新词频
        new_word_freqs = Counter()
        for word, freq in word_freqs.items():
            new_word = word.replace(' '.join(best_pair), new_token)
            new_word_freqs[new_word] = freq
        word_freqs = new_word_freqs
    
    return vocab, merges
```

### 3.3 编码

```python
def encode(text, merges):
    # 分词
    words = text.split()
    
    tokens = []
    for word in words:
        # 字符级开始
        word_tokens = list(word) + ['</w>']
        
        # 应用合并规则
        for pair in merges:
            i = 0
            while i < len(word_tokens) - 1:
                if (word_tokens[i], word_tokens[i+1]) == pair:
                    word_tokens = word_tokens[:i] + [pair[0]+pair[1]] + word_tokens[i+2:]
                else:
                    i += 1
        
        tokens.extend(word_tokens)
    
    return tokens
```

### 3.4 示例

```
训练语料: "hug hug pug hug pug"

初始词表: h, u, g, p, </w>

统计对:
(h, u): 3
(u, g): 5
(g, </w>): 5
...

合并 (u, g) → ug:
词表: h, u, g, p, </w>, ug

继续合并...
```

## 4. WordPiece

### 4.1 与 BPE 的区别

BPE：合并频率最高的对

WordPiece：合并能最大化似然的字符对

```
Score = P(AB) / (P(A) × P(B))
```

选择使得合并后似然增加最多的对。

### 4.2 实现

```python
def train_wordpiece(texts, vocab_size):
    # 初始化
    word_freqs = Counter()
    for text in texts:
        for word in text.split():
            word_freqs[word] += 1
    
    # 初始词表（字符）
    vocab = set()
    for word in word_freqs:
        vocab.update(list(word))
    
    while len(vocab) < vocab_size:
        # 计算每个对得分
        pair_scores = {}
        for word, freq in word_freqs.items():
            chars = list(word)
            for i in range(len(chars) - 1):
                pair = (chars[i], chars[i+1])
                # 计算得分
                score = freq / (char_freq[chars[i]] * char_freq[chars[i+1]])
                pair_scores[pair] = pair_scores.get(pair, 0) + score
        
        # 选择最高分对
        best_pair = max(pair_scores, key=pair_scores.get)
        new_token = best_pair[0] + best_pair[1]
        vocab.add(new_token)
        
        # 更新
        # ...
    
    return vocab
```

### 4.3 特点

BERT 使用 WordPiece：

```
"unwanted" → ["un", "##want", "##ed"]
```

`##` 表示这是一个子词（接在前面的词后面）。

## 5. SentencePiece

### 5.1 为什么需要 SentencePiece？

传统方法：
- 需要先分词（按空格）
- 不同语言分词规则不同

SentencePiece：
- 直接处理原始文本
- 不需要预分词
- 语言无关

### 5.2 实现

```python
import sentencepiece as spm

# 训练
spm.SentencePieceTrainer.train(
    input='corpus.txt',
    model_prefix='m',
    vocab_size=32000,
    model_type='bpe'  # 或 'unigram'
)

# 加载
sp = spm.SentencePieceProcessor()
sp.load('m.model')

# 编码
text = "我喜欢学习"
tokens = sp.encode(text, out_type=str)
ids = sp.encode(text, out_type=int)

# 解码
decoded = sp.decode(ids)
```

### 5.3 特点

- 支持 BPE 和 Unigram
- 语言无关
- 支持多语言
- 可逆编码

## 6. 现代分词器

### 6.1 GPT 系列

使用 **Byte-level BPE**：

```python
from transformers import GPT2Tokenizer

tokenizer = GPT2Tokenizer.from_pretrained('gpt2')

text = "Hello, world!"
tokens = tokenizer.encode(text)
# [15496, 11, 995, 0]

decoded = tokenizer.decode(tokens)
# "Hello, world!"
```

特点：
- 字节级（256 个基础 token）
- 处理所有 Unicode 字符
- 词表大小通常 50k

### 6.2 LLaMA

使用 SentencePiece + BPE：

```python
from transformers import LlamaTokenizer

tokenizer = LlamaTokenizer.from_pretrained('llama-7b')

text = "Hello, world!"
tokens = tokenizer.encode(text)
```

特点：
- 更高效的编码
- 支持多语言
- 32000 词表

### 6.3 对比

| 模型 | 分词器 | 词表大小 | 特点 |

| GPT-2 | Byte BPE | 50,257 | 字节级，通用 |

## 8. 数据清洗

### 8.1 为什么需要清洗？

原始文本质量问题：
- 重复内容
- 低质量文本
- 有害内容
- 格式混乱

### 8.2 常见清洗步骤

```python
import re
from bs4 import BeautifulSoup

def clean_text(text):
    # 1. 去除 HTML 标签
    text = BeautifulSoup(text, 'html.parser').get_text()
    
    # 2. 统一换行符
    text = text.replace('\r\n', '\n').replace('\r', '\n')
    
    # 3. 去除多余空白
    text = re.sub(r'[ \t]+', ' ', text)
    text = re.sub(r'\n{3,}', '\n\n', text)
    
    # 4. 去除特殊字符（可选）
    # text = re.sub(r'[^\w\s\u4e00-\u9fff，。！？、；：""''（）]', '', text)
    
    # 5. 去除 URL（可选）
    text = re.sub(r'https?://\S+', '', text)
    
    # 6. 去除邮箱（可选）
    text = re.sub(r'\S+@\S+', '', text)
    
    # 7. strip
    text = text.strip()
    
    return text
```

### 8.3 去重

```python
def deduplicate(texts, threshold=0.8):
    """
    使用 MinHash 或 SimHash 去重
    """
    from datasketch import MinHash, MinHashLSH
    
    lsh = MinHashLSH(threshold=threshold, num_perm=128)
    
    unique_texts = []
    for i, text in enumerate(texts):
        # 创建 MinHash
        mh = MinHash(num_perm=128)
        for word in text.split():
            mh.update(word.encode('utf-8'))
        
        # 检查是否重复
        if not lsh.query(mh):
            lsh.insert(str(i), mh)
            unique_texts.append(text)
    
    return unique_texts
```

### 8.4 质量过滤

```python
def filter_quality(texts):
    filtered = []
    for text in texts:
        # 1. 长度过滤
        if len(text) < 100 or len(text) > 100000:
            continue
        
        # 2. 语言检测
        # if not is_target_language(text):
        #     continue
        
        # 3. 词汇多样性
        words = text.split()
        if len(set(words)) / len(words) < 0.3:  # 重复率高
            continue
        
        # 4. 特殊字符比例
        special_char_ratio = sum(1 for c in text if not c.isalnum() and c not in ' \n') / len(text)
        if special_char_ratio > 0.3:
            continue
        
        # 5. 有害内容检测
        # if contains_harmful_content(text):
        #     continue
        
        filtered.append(text)
    
    return filtered
```

### 8.5 GPT 系列的数据处理

```python
def prepare_gpt_data(raw_texts):
    # 1. 清洗
    cleaned = [clean_text(text) for text in raw_texts]
    
    # 2. 过滤
    filtered = filter_quality(cleaned)
    
    # 3. 去重
    deduped = deduplicate(filtered)
    
    # 4. 分词
    tokenized = []
    for text in deduped:
        tokens = tokenizer.encode(text)
        tokenized.append(tokens)
    
    # 5. 切分成长度适中的块
    max_length = 1024
    chunks = []
    for tokens in tokenized:
        for i in range(0, len(tokens), max_length):
            chunk = tokens[i:i+max_length]
            if len(chunk) >= 100:  # 过滤太短的
                chunks.append(chunk)
    
    return chunks
```

## 9. 数据集构建

### 9.1 Dataset 类

```python
from torch.utils.data import Dataset

class TextDataset(Dataset):
    def __init__(self, texts, tokenizer, max_length=1024):
        self.tokenizer = tokenizer
        self.max_length = max_length
        self.examples = []
        
        for text in texts:
            tokens = tokenizer.encode(text)
            
            # 按固定长度切分
            for i in range(0, len(tokens) - max_length, max_length):
                chunk = tokens[i:i+max_length+1]
                self.examples.append(chunk)
    
    def __len__(self):
        return len(self.examples)
    
    def __getitem__(self, idx):
        tokens = self.examples[idx]
        
        input_ids = torch.tensor(tokens[:-1])
        labels = torch.tensor(tokens[1:])
        
        return {
            'input_ids': input_ids,
            'labels': labels
        }
```

### 9.3 内存映射

对于大数据集，使用内存映射：

```python
import numpy as np

def save_to_memmap(tokens_list, filename):
    # 计算总长度
    total_length = sum(len(tokens) for tokens in tokens_list)
    
    # 创建内存映射文件
    arr = np.memmap(filename, dtype=np.int32, mode='w+', shape=(total_length,))
    
    # 写入
    idx = 0
    for tokens in tokens_list:
        arr[idx:idx+len(tokens)] = tokens
        idx += len(tokens)
    
    return arr

class MemmapDataset(Dataset):
    def __init__(self, filename, total_length, max_length=1024):
        self.data = np.memmap(filename, dtype=np.int32, mode='r', shape=(total_length,))
        self.max_length = max_length
    
    def __len__(self):
        return len(self.data) - self.max_length
    
    def __getitem__(self, idx):
        chunk = self.data[idx:idx+self.max_length+1]
        return {
            'input_ids': torch.tensor(chunk[:-1]),
            'labels': torch.tensor(chunk[1:])
        }
```

## 11. 常见问题

### Q1: 词表大小怎么选？

经验值：
- 小模型：10k-30k
- 中模型：30k-50k
- 大模型：50k-100k

权衡：
- 太小：序列长，推理慢
- 太大：稀疏，训练难

### Q2: BPE vs WordPiece vs Unigram？

| 方法 | 优点 | 缺点 |
|------|------|------|
| BPE | 简单高效 | 可能合并不合理的对 |
| WordPiece | 更语言学 | 训练慢 |
| Unigram | 概率模型 | 实现复杂 |

GPT 系列：Byte BPE
BERT 系列：WordPiece
LLaMA：SentencePiece

### Q3: 如何处理多语言？

- 用 SentencePiece
- 字节级 BPE
- 大词表（100k+）
- 语言标记

### Q4: 中文分词的特殊性？

中文没有空格，需要：
- 字符级（简单）
- 分词器（结巴、pkuseg）
- SentencePiece（推荐）

## 12. 核心要点总结

```
分词：文本 → 数字

方法：
- 词级：词表大，有 OOV
- 字符级：词表小，序列长
- 子词级：平衡，主流

主流算法：
- BPE：合并高频对
- WordPiece：最大化似然
- SentencePiece：语言无关

数据处理：
- 清洗：去除噪声
- 过滤：保证质量
- 去重：避免重复
- 分词：转为数字

词表大小：
- 小模型：10k-30k
- 中模型：30k-50k
- 大模型：50k-100k
```
