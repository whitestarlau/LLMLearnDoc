# 图神经网络：非欧几里得数据的处理

## 历史发展脉络

### 早期探索：递归神经网络与图（1990s）

图神经网络的思想可追溯到将神经网络应用于结构化数据的早期尝试：

**递归神经网络阶段**
- **1997年**：Sperduti和Starita首次将递归神经网络应用于有向无环图（DAG）
- **1998年**：Frasconi等人进一步发展，用于处理树结构和图结构数据
- **特点**：只能处理无环图，存在收敛性问题

**早期GNN概念的提出**
- **2005年**：Marco Gori等人在论文中首次提出"图神经网络"（Graph Neural Network）的概念
- **2009年**：Franco Scarselli等人在IEEE TNN期刊发表奠基性论文《The Graph Neural Network Model》
- **起源地**：意大利锡耶纳大学
- **核心思想**：通过迭代状态更新，让每个节点从邻居获取信息直到收敛
- **局限**：计算开销大，收敛速度慢，难以扩展到大型图

### 谱域图卷积的兴起（2013-2016）

受卷积神经网络（CNN）在图像处理成功的启发，研究者试图将卷积概念推广到图数据：

**谱图卷积网络（2013）**
- **创始人**：Joan Bruna等（纽约大学）
- **论文**：《Spectral Networks and Locally Connected Networks on Graphs》
- **背景**：试图将CNN的局部感受野思想推广到非欧几里得数据
- **方法**：基于图傅里叶变换定义卷积操作
- **局限**：需要计算拉普拉斯矩阵的特征分解，计算复杂度O(n³)

**ChebNet（2016）**
- **创始人**：Michaël Defferrard等（洛桑联邦理工学院）
- **论文**：《Convolutional Neural Networks on Graphs with Fast Localized Spectral Filtering》
- **创新**：使用切比雪夫多项式近似谱卷积，避免特征分解
- **意义**：首次实现高效的局部化图卷积

**GCN的诞生（2016年9月）**
- **创始人**：Thomas Kipf、Max Welling（阿姆斯特丹大学）
- **论文**：《Semi-Supervised Classification with Graph Convolutional Networks》
- **背景**：
  - ChebNet虽然高效但参数较多
  - 需要一种更简单、可扩展的图卷积方法
- **核心创新**：
  - 一阶切比雪夫近似，简化为简单的邻居聚合
  - 对称归一化避免数值不稳定
  - 仅需一个权重矩阵，参数效率高
- **公式**：H⁽ˡ⁺¹⁾ = σ(D^(-1/2) Â D^(-1/2) H⁽ˡ⁾ W⁽ˡ⁾)
- **影响**：成为图神经网络的代名词，引用量超过25000次

### 空间方法与注意力机制（2017-2018）

**GraphSAGE（2017年）**
- **创始人**：William Hamilton等（斯坦福大学）
- **论文**：《Inductive Representation Learning on Large Graphs》
- **背景**：GCN是直推式方法，无法处理新节点
- **核心创新**：
  - **采样聚合**：对邻居采样后聚合，支持小批量训练
  - **归纳学习**：学习聚合函数而非固定嵌入，可泛化到新节点
  - **多种聚合器**：Mean、LSTM、Pooling
- **解决问题**：大规模图的训练效率和归纳学习能力

**图注意力网络GAT（2017年10月）**
- **创始人**：Petar Veličković等（剑桥大学、Yoshua Bengio实验室）
- **论文**：《Graph Attention Networks》（ICLR 2018）
- **背景**：
  - GCN对所有邻居一视同仁，无法区分重要性
  - 需要一种能学习邻居权重的机制
- **核心创新**：
  - 将Transformer的自注意力机制引入图神经网络
  - 每个节点对邻居学习不同的注意力权重
  - 多头注意力提高表达能力
- **优势**：
  - 自动学习邻居重要性，可解释性强
  - 不依赖全局图结构，适合归纳学习
  - 计算高效，可并行化

### 几何深度学习的统一视角（2017）

**奠基性综述**
- **作者**：Michael M. Bronstein等
- **论文**：《Geometric Deep Learning: Going Beyond Euclidean Data》（IEEE SPM 2017）
- **意义**：
  - 首次系统统一了图神经网络的理论框架
  - 提出"几何深度学习"概念，涵盖图、流形、群等非欧数据
  - 为后续研究提供了清晰的分类体系

### 消息传递范式的统一（2017）

**Gilmer et al.的消息传递框架**
- **论文**：《Neural Message Passing for Quantum Chemistry》
- **贡献**：统一描述了GCN、GraphSAGE、GAT等方法为"消息传递神经网络"（MPNN）
- **框架**：消息函数 → 聚合函数 → 更新函数
- **影响**：成为图神经网络的通用编程范式

### 进阶发展（2019-至今）

**图Transformer（2021-）**
- **Graphormer**（2021）：微软亚洲研究院，首次将Transformer成功应用于图
- **GPS**（2022）：通用图Transformer，结合局部消息传递和全局注意力
- **背景**：探索Transformer架构在图上的表达能力

**异构图与动态图**
- **R-GCN**（2017）：处理多种关系类型的图
- **EvolveGCN**（2020）：处理随时间演化的图
- **TGAT**（2020）：时间图注意力网络

**表达能力的理论研究**
- **GIN**（2019）：图同构网络，证明最大区分能力
- **WL测试**：将GNN的表达能力与Weisfeiler-Leman图同构测试联系起来

### 当代应用与前沿

## 为什么需要图神经网络

传统的深度学习方法（如CNN、RNN）擅长处理规则数据（图像、序列），但现实世界中很多数据具有图结构：

- 社交网络：用户及其关系
- 分子结构：原子及其化学键
- 知识图谱：实体及其关系
- 交通网络：道路及其连接

### 图数据的特点

1. **不规则结构**：节点数量和连接方式可变
2. **无序性**：节点没有天然的顺序
3. **局部连接**：每个节点只与部分邻居相连

## 图神经网络基础

### 图的表示

- **节点（Node）**：图中的实体
- **边（Edge）**：节点之间的关系
- **节点特征**：每个节点的属性向量
- **邻接矩阵**：表示节点间连接关系

### 消息传递范式

图神经网络的核心是消息传递（Message Passing）：

1. **消息（Message）**：邻居节点发送信息
2. **聚合（Aggregate）**：汇总邻居信息
3. **更新（Update）**：更新节点表示

```
h_v^(k+1) = Update(h_v^(k), Aggregate({Message(h_u^(k)) : u ∈ N(v)}))
```

## 经典图神经网络

### 图卷积网络（GCN）

GCN 是最经典的图神经网络，基于图上的卷积操作。

**传播规则**
```
H^(l+1) = σ(D^(-1/2) Â D^(-1/2) H^(l) W^(l))
```

其中：
- Â = A + I（加入自环的邻接矩阵）
- D：度矩阵
- H：节点特征矩阵
- W：可学习权重

**特点**
- 局部卷积：只聚合邻居信息
- 谱域解释：基于图傅里叶变换

### GraphSAGE

采样聚合方法，适用于大规模图：

```
h_v = σ(W · CONCAT(h_v, AGGREGATE({h_u : u ∈ N(v)})))
```

**聚合函数**
- Mean：邻居特征均值
- LSTM：序列化聚合
- Pooling：最大池化

### 图注意力网络（GAT）

引入注意力机制，学习邻居的重要性：

```
α_ij = softmax(LeakyReLU(a^T [Wh_i || Wh_j]))
h_i' = σ(∑_j α_ij W h_j)
```

**优势**
- 自动学习邻居权重
- 可解释性强

## 进阶图神经网络

### 异构图神经网络

处理多种类型节点和边的图：

- **R-GCN**：关系图卷积网络，为每种关系类型使用不同权重
- **HAN**：层次注意力网络，节点级和语义级注意力

### 动态图神经网络

处理随时间变化的图：

- **EvolveGCN**：使用RNN演化GCN参数
- **TGAT**：时间图注意力网络

### 图Transformer

将Transformer架构应用于图：

- **Graphormer**：使用中心性编码和空间编码
- **GPS**：通用图Transformer

## 实现示例

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features)
    
    def forward(self, x, adj):
        # x: 节点特征 [N, in_features]
        # adj: 归一化邻接矩阵 [N, N]
        x = self.linear(x)
        x = torch.mm(adj, x)  # 邻居聚合
        return F.relu(x)

class GCN(nn.Module):
    def __init__(self, n_features, n_classes):
        super().__init__()
        self.conv1 = GCNLayer(n_features, 64)
        self.conv2 = GCNLayer(64, n_classes)
    
    def forward(self, x, adj):
        x = self.conv1(x, adj)
        x = self.conv2(x, adj)
        return F.log_softmax(x, dim=1)
```
