# 07-07 多模态：感知世界的多种方式

上一章我们深入了解了 Diffusion 模型——这项技术让 AI 能够从纯粹的噪声中生成精美的图像。但 Diffusion 只是单模态的生成：输入可能是文本，输出是图像，模型本身仍然只"看到"图像这一种信息。

**现实世界远比这复杂**。我们人类感知世界从来不是通过单一渠道：我们看到物体的形状和颜色，听到它们发出的声音，触摸它们的质地，甚至闻到它们的气味。这些不同形式的感官信息在我们的大脑中融合，形成对世界的完整理解。

AI 也正在朝这个方向进化。从 2023 年的 GPT-4V 能够"看懂"图片，到 2024 年的 GPT-4o 能够实时处理音频输入和输出，再到 Gemini 原生支持图文音三种模态——**多模态大模型正在打破单一模态的边界**。

本章将聚焦多模态学习的核心：如何让 AI 同时理解图像、文本和音频等多种信息形式，通过对齐不同模态的语义空间，实现真正的跨模态理解。

---

## 1. 多模态学习的本质

### 1.1 什么是模态？

**模态（Modality）** 就是信息的表现形式。我们人类感知世界的方式天然是多模态的：

| 感官 | 信息形式 | AI中的模态 |
|------|----------|-----------|
| 视觉 | 光线、颜色、形状 | **图像**、**视频** |
| 听觉 | 声音、语调、音乐 | **音频** |
| 语言 | 文字、语义 | **文本** |
| 触觉 | 质地、温度、压力 | 触觉传感器数据 |
| 嗅觉 | 气味分子 | 化学传感器数据 |

在当前的AI系统中，**文本、图像和音频**是三种最主要的模态。多模态学习的目标，就是让AI能够像人类一样，同时处理和理解这些不同形式的信息。

### 1.2 音频模态：从视觉到听觉的拓展

2024年，GPT-4o 的发布标志着一个重要的转变：大模型不再局限于"看懂"图片，还能"听懂"声音、"说出"语言。这得益于音频模态的引入。

**音频模态的独特价值**：

| 应用场景 | 说明 | 示例 |
|----------|------|------|
| **语音交互** | 直接用声音与AI对话 | "播放轻音乐，我要工作了" |
| **声音识别** | 识别环境中的声音 | "这段音频里有救护车鸣笛" |
| **音乐理解** | 分析音乐的情感和风格 | "这首曲子适合什么场合？" |
| **多语言支持** | 听懂不同语言的口语 | 直接说中文、英文、日文 |
| **情感感知** | 从语调中判断情绪 | "用户听起来很着急" |

**音频处理的挑战**：

```
图像：2D矩阵 (H × W × 3)
文本：1D序列 (token序列)
音频：1D波形 (高采样率，每秒数万个样本)

音频的难点：
- 信息密度高：1秒音频包含大量波形点
- 时间连续：不像文本有明确的分词边界
- 多维度：包含语义内容、说话人特征、情感、环境音等
```

**解决方案**：将音频转换为频谱图（Spectrogram）

```
原始音频波形 → 傅里叶变换 → 频谱图（可视为图像）
                 ↓
         用类似ViT的编码器处理
```

现代多模态模型（如 GPT-4o、Gemini）通常将音频编码为特殊的"音频token"，与文本token一起输入Transformer进行处理。

### 1.3 为什么需要多模态？

单模态AI的局限：

```
纯文本模型：只能处理文字，无法"看见"或"听见"世界
  → 问："这张图片里有什么？" → 无法回答
  → 问："这段音频说了什么？" → 无法回答
  
纯视觉模型：只能处理图像，无法"理解"语义或"听到"声音
  → 给图像配上准确的文字描述 → 困难
  → 分析视频中的对话内容 → 不可能
```

多模态的价值：

| 应用场景 | 单模态 | 多模态 |
|----------|--------|--------|
| 看图回答 | ❌ | ✅ "图片中有一只橘猫" |
| 听音识意 | ❌ | ✅ "音频里有人在讨论天气" |
| 视频理解 | 只能看画面或只听声音 | ✅ 同时理解画面和对话 |
| AI助手 | 纯文字对话 | ✅ 看到屏幕、听到声音、实时对话 |
| 内容创作 | 只能写文字 | ✅ 可以配图、配音、做视频 |

### 1.4 多模态的核心挑战

**不同模态之间的"翻译"**

```
图像 → 像素矩阵 → 高维向量
文本 → 词嵌入 → 高维向量

问题：这两个向量的空间不一样！
```

**解决方案**：学习一个共同的"语义空间"

```
图像encoder → 图像向量
                ↓ 对齐到同一空间
文本encoder  → 文本向量
```

在这个空间里，"猫"的图像和"猫"的文本应该非常接近！

## 2. 经典方法与模型

### 2.1 CLIP：连接图像与文本的桥梁

**CLIP（Contrastive Language-Image Pre-training）** 是多模态领域最重要的突破之一。

**核心思想**：对比学习

```
训练数据：(图像, 文本描述)配对
- 一张图片 + 正确的描述 → 拉近距离
- 一张图片 + 错误的描述 → 推远距离
```

**训练目标**：

$$\mathcal{L} = -\log \frac{\exp(\text{sim}(I, T)/\tau)}{\sum_{j=1}^{N} \exp(\text{sim}(I, T_j)/\tau)}$$

其中 $\text{sim}$ 是余弦相似度，$\tau$ 是温度参数。

**CLIP的能力**：

- **零样本分类**：没见过的图像也能分类
- **图文匹配**：判断图像和文本是否匹配
- **跨模态检索**：用文字搜图片，用图片搜文字

### 2.2 视觉语言模型（VLM）

**架构演进**：

```
第一代：冻结视觉编码器 + 训练投影层
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 图像    │ →  │ ViT     │ →  │ 投影层  │ →  LLM
│ 输入    │    │ 编码器  │    │ (线性)  │    (冻结)
└─────────┘    └─────────┘    └─────────┘

第二代：视觉编码器参与训练
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 图像    │ →  │ ViT     │ →  │ 投影层  │ →  LLM
│ 输入    │    │ 编码器  │    │ (Q-Form)│    (微调)
└─────────┘    └─────────┘    └─────────┘

第三代：端到端多模态
┌─────────┐    ┌─────────────────┐
│ 图像    │ →  │ 统一Transformer │
│ 文本    │ →  │ (视觉+语言融合) │
└─────────┘    └─────────────────┘
```

### 2.3 著名多模态模型

| 模型 | 支持模态 | 特点 | 发布时间 |
|------|---------|------|----------|
| **GPT-4V** | 图+文 | OpenAI最强视觉理解，分析图片和图表 | 2023 |

### 2.4 按文生图：多模态生成

**Stable Diffusion + CLIP**：

```
文本输入 → CLIP文本编码器 → 文本向量
                                    ↓ 交叉注意力
随机噪声 → UNet去噪 ←───────────────┘
                ↓ 逐步去噪
             生成图像
```

**训练流程**：

```
Step 1: 文本编码器 (CLIP) - 学会理解文本
Step 2: UNet - 学会根据文本向量去噪
Step 3: VAE解码器 - 学会从潜空间重建图像
```

## 3. 数学原理

### 3.1 对比学习损失函数

对于batch大小为 $N$ 的数据：

**图像到文本的损失**：

$$L_{I2T} = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(\mathbf{I}_i \cdot \mathbf{T}_i / \tau)}{\sum_{j=1}^{N} \exp(\mathbf{I}_i \cdot \mathbf{T}_j / \tau)}$$

**文本到图像的损失**：

$$L_{T2I} = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(\mathbf{T}_i \cdot \mathbf{I}_i / \tau)}{\sum_{j=1}^{N} \exp(\mathbf{T}_i \cdot \mathbf{I}_j / \tau)}$$

**总损失**：

$$\mathcal{L} = \frac{L_{I2T} + L_{T2I}}{2}$$

### 3.2 跨模态注意力

在视觉语言模型中，核心是跨模态注意力：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中：
- $Q$ 来自文本（语言查询）
- $K, V$ 来自图像（视觉键值）

**多层交叉注意力**：

```
文本特征 → Linear → Q
图像特征 → Linear → K, V

多头注意力 → 输出融合表示
```

### 3.3 投影层设计

最简单的投影是线性层：

$$\mathbf{h}_{vis} = W_{proj} \cdot \mathbf{h}_{image}$$

更复杂的Q-Former（BLIP-2中使用）：

```
图像向量 → Query Queries
               ↓ 交叉注意力 ← 文本向量
            输出融合向量
```

## 4. 重要技术

### 4.1 视觉编码器选择

| 编码器 | 参数量 | 特点 |
|--------|--------|------|
| **ViT-B** | 86M | 小型快速 |
| **ViT-L** | 307M | 平衡选择 |
| **ViT-G** | 1.8B | 效果最好 |
| **SigLIP** | 400M | 更稳定的对比学习 |
| **DINOv2** | 1B | 强大的视觉表示 |

### 4.2 分辨率与切片

高分辨率图像处理：

```
原始图像: 1024x1024
     ↓ 切分成 4 个 512x512
     ↓ 分别编码
     ↓ 拼接到一起
```

**好处**：保持细节，处理任意尺寸

### 4.3 训练策略

| 策略 | 做法 | 优点 |
|------|------|------|
| **冻结视觉** | 只训练投影层 | 省显存，快 |
| **部分冻结** | 冻结底层，微调上层 | 平衡 |
| **完全微调** | 所有参数参与训练 | 效果最好 |
| **LoRA** | 只训练低秩矩阵 | 高效微调 |

### 4.4 数据构建

高质量多模态数据的关键：

```
图像描述对：
- alt-text（网页图片的描述）
- 人工标注描述
- BLIP生成 + 人工过滤

指令数据：
- VQA（视觉问答）
- 图像描述任务
- 复杂推理任务
```

## 5. 代码实现

### 5.1 CLIP模型使用

```python
import torch
from transformers import CLIPProcessor, CLIPModel

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

from PIL import Image

image = Image.open("cat.jpg")
texts = ["a photo of a cat", "a photo of a dog", "a photo of a car"]

inputs = processor(text=texts, images=image, return_tensors="pt", padding=True)

outputs = model(**inputs)
logits_per_image = outputs.logits_per_image
probs = logits_per_image.softmax(dim=1)

print(f"分类概率: {probs}")
```

### 5.2 简易多模态对话（使用LLaVA思路）

```python
import torch
from transformers import AutoModel, AutoProcessor

class SimpleVLM(torch.nn.Module):
    def __init__(self, vision_model_name="google/siglip-base-patch16-224", 
                 llm_name="microsoft/phi-2"):
        super().__init__()
        self.vision_encoder = AutoModel.from_pretrained(vision_model_name)
        self.projection = torch.nn.Linear(768, 2048)
        self.llm = AutoModel.from_pretrained(llm_name)
    
    def forward(self, pixel_values, input_ids, attention_mask):
        vision_outputs = self.vision_encoder.vision_model(pixel_values)
        image_embeds = vision_outputs.last_hidden_state
        image_embeds = self.projection(image_embeds)
        
        llm_outputs = self.llm(
            input_ids=input_ids,
            attention_mask=attention_mask,
            pixel_values=pixel_values,
            image_embeds=image_embeds
        )
        return llm_outputs.logits

vl_model = SimpleVLM()
vl_model = vl_model.cuda()
vl_model.train()

optimizer = torch.optim.AdamW(vl_model.parameters(), lr=1e-5)

for batch in dataloader:
    pixel_values = batch["pixel_values"].cuda()
    input_ids = batch["input_ids"].cuda()
    labels = batch["labels"].cuda()
    
    logits = vl_model(pixel_values, input_ids, attention_mask=None)
    loss = torch.nn.functional.cross_entropy(
        logits.view(-1, logits.size(-1)), labels.view(-1)
    )
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

### 5.3 图像描述生成

```python
from transformers import BlipProcessor, BlipForConditionalGeneration

processor = BlipProcessor.from_pretrained("Salesforce/blip-image-captioning-base")
model = BlipForConditionalGeneration.from_pretrained("Salesforce/blip-image-captioning-base")

image = Image.open("landscape.jpg")

inputs = processor(image, return_tensors="pt")

out = model.generate(**inputs, max_new_tokens=50)
caption = processor.decode(out[0], skip_special_tokens=True)

print(f"图像描述: {caption}")
```

## 6. 总结

多模态AI的关键里程碑：

1. **CLIP（2021）**：开创性地用对比学习连接图像和文本
2. **BLIP-2（2023）**：引入Q-Former，实现高效视觉语言对齐
3. **LLaVA（2023）**：开源方案，验证指令微调的有效性
4. **GPT-4V（2023）**：展示强大的多模态理解能力

**为什么多模态重要**：

- **更自然的人机交互**：可以看图、可以说话、可以听声音
- **更强大的智能**：文本、图像、音频互补，信息更丰富
- **更广泛的应用**：医疗影像+病历、视频理解、智能语音助手、教育、设计、娱乐...

**未来方向**：

- 原生多模态大模型（不是拼接）
- 视频理解与生成
- 3D场景理解
- 具身智能（多模态+机器人）

多模态正是让AI走向"通用智能"的关键一步！
